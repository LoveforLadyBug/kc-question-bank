4 q003 Chapter 02 — BCS 난이도 이해

해설

카카오클라우드 공식 문서에 따르면, BCS 인스턴스 명명 규칙은 인스턴스 패밀리, 세대, 프로세서, 옵션, 크기를 조합합니다. c2a.xlarge를 분석하면 다음과 같습니다. 'c'는 컴퓨팅 최적화(Compute optimized) 패밀리, '2'는 2세대, 'a'는 AMD 프로세서, '.xlarge'는 인스턴스 크기를 의미합니다. 따라서 c2a.xlarge는 컴퓨팅 최적화 유형의 2세대 AMD 프로세서 기반 엑스트라 라지 인스턴스입니다.

오답 포인트
  • A: 메모리 최적화 인스턴스 패밀리는 'r'로 시작합니다(예: r2a). 'c'는 컴퓨팅 최적화를 의미합니다.
  • C: 범용 인스턴스 패밀리는 'm'으로 시작합니다(예: m2a, m3az). 'c'는 컴퓨팅 최적화를 의미합니다.
  • D: 가속 컴퓨팅 중 FPGA 관련 인스턴스는 'gf'로 시작합니다(예: gf1i). 'c'는 컴퓨팅 최적화 패밀리를 의미합니다.

6 q002 Chapter 02 — BCS 난이도 기본

카카오클라우드 Bare Metal Server(BMS)에서 현재 제공하는 인스턴스 유형으로 옳은 것은?

해설

카카오클라우드 공식 문서에 따르면, Bare Metal Server(BMS)는 메모리 최적화(Memory optimized)와 가속 컴퓨팅(Accelerated computing) 인스턴스 유형만 제공합니다. 범용(General purpose) 인스턴스는 BMS에서 추후 지원 예정으로 현재는 제공되지 않으며, 컴퓨팅 최적화(Compute optimized) 인스턴스는 VM에서만 제공됩니다.

오답 포인트
  • A: 범용 인스턴스는 BMS에서 추후 지원 예정이며 현재는 제공되지 않습니다. 컴퓨팅 최적화 인스턴스도 BMS에서는 제공되지 않습니다.
  • C: 범용 인스턴스는 BMS에서 현재 제공되지 않습니다. BMS는 고성능이 요구되는 메모리 최적화와 가속 컴퓨팅 유형만 제공합니다.
  • D: 컴퓨팅 최적화(c2a) 인스턴스는 VM에서만 제공되며, BMS에서는 제공되지 않습니다.
7 q002 Chapter 02 — BNS 난이도 이해

카카오클라우드에서 VPC를 생성할 때 자동으로 함께 생성되는 리소스로 옳은 것은?

해설

카카오클라우드 공식 문서에 따르면, 카카오클라우드에서 생성되는 모든 VPC는 생성 시 기본 서브넷, 연결된 인터넷 게이트웨이, 모든 트래픽을 인터넷 게이트웨이로 보내는 기본 라우팅 테이블의 라우팅까지 함께 생성됩니다. 가용 영역에 생성된 기본 서브넷에 BCS 인스턴스를 생성하고 퍼블릭 IP를 연결하면 바로 인터넷에 액세스할 수 있습니다.

오답 포인트
  • A: 퍼블릭 IP는 VPC 생성 시 자동으로 생성되지 않습니다. 퍼블릭 IP는 별도로 생성하여 네트워크 인터페이스에 연결해야 합니다.
  • C: NAT 게이트웨이는 VPC 생성 시 자동으로 생성되지 않습니다. 프라이빗 서브넷에서 인터넷에 액세스하려면 NAT 인스턴스를 별도로 구성해야 합니다.
  • D: 로드 밸런서는 VPC 생성 시 자동으로 생성되지 않습니다. 트래픽 분산이 필요한 경우 Load Balancing 서비스를 별도로 사용해야 합니다.

카카오클라우드 Alert Center에서 설정 가능한 알림 정책의 이벤트 조건 유형을 모두 올바르게 나열한 것은?

해설

카카오클라우드 공식 문서에 따르면, Alert Center에서는 메트릭(Metric), 로그(Log), 이벤트(Event) 세 가지 유형으로 알림 정책을 설정할 수 있습니다. 메트릭 기반은 CPU/메모리/디스크 등 시스템 성능 알림, 로그 기반은 특정 키워드 발생 알림, 이벤트 기반은 서비스별 특화 이벤트(리소스 생성/삭제 등) 알림입니다.

오답 포인트
  • A: 메트릭과 로그만 나열되어 있습니다. Alert Center는 이벤트(Event) 유형도 지원하므로 세 가지 모두 포함해야 합니다.
  • B: 메트릭과 이벤트만 나열되어 있습니다. 로그(Log) 기반 알림 정책도 지원하므로 불완전한 답입니다.
  • D: 트레이스(Trace)는 Alert Center 알림 정책의 이벤트 조건 유형에 포함되지 않습니다. 지원 유형은 메트릭, 로그, 이벤트입니다.

10 q002 Chapter 02 — Container Pack 난이도 이해

카카오클라우드 Kubernetes Engine의 클러스터 구성에 대한 설명으로 옳은 것은?

해설

카카오클라우드 공식 문서에 따르면, Kubernetes Engine의 제어 영역(Control Plane)은 카카오클라우드가 관리하는 별도의 VPC에서 완전 관리형으로 운영됩니다. 워커 노드는 사용자의 VPC에 배포되며, 사용자는 워커 노드와 그 위에서 실행되는 애플리케이션에만 집중하면 됩니다.

오답 포인트
  • A: 제어 영역은 사용자 VPC가 아닌, 카카오클라우드가 관리하는 별도의 VPC에 위치합니다. 사용자 VPC에는 워커 노드만 배포됩니다.
  • C: 제어 영역은 사용자가 직접 설정하는 것이 아니라 카카오클라우드가 완전 관리합니다. 관리형 서비스의 핵심 특징입니다.
  • D: 제어 영역은 카카오클라우드의 별도 VPC에 있고 워커 노드는 사용자 VPC에 있으므로, 동일한 서브넷에 위치한다는 설명은 잘못되었습니다.
11 q004 Chapter 02 — BNS 난이도 기본

카카오클라우드에서 퍼블릭 서브넷과 프라이빗 서브넷을 구분하는 기준으로 옳은 것은?

해설

카카오클라우드 공식 문서에 따르면, 퍼블릭 서브넷은 연결된 라우팅 테이블에 인터넷 게이트웨이로의 라우팅(Route) 설정이 있는 서브넷입니다. 이 경로를 통해 퍼블릭 서브넷 상의 리소스는 인터넷에 접근할 수 있습니다. 프라이빗 서브넷은 연결된 라우팅 테이블에 인터넷 게이트웨이로의 라우팅 설정이 없는 서브넷이며, 프라이빗 서브넷의 리소스가 인터넷에 액세스하려면 NAT 인스턴스가 필요합니다.

오답 포인트
  • A: 카카오클라우드 서브넷의 IPv4 주소 범위는 서브넷 유형(퍼블릭/프라이빗)에 관계없이 항상 프라이빗 IP입니다. 즉, IP 대역으로 서브넷 유형을 구분하지 않습니다.
  • C: 보안 그룹은 퍼블릭/프라이빗 서브넷 구분과 관계없이 인스턴스 보안을 위해 적용되는 것입니다. 서브넷 유형 기준이 아닙니다.
  • D: 인스턴스 배포 여부는 서브넷 유형(퍼블릭/프라이빗)과 관계가 없습니다. 두 유형 모두 인스턴스를 배포할 수 있습니다.
12 q001 Chapter 03 — Billing 난이도 이해

카카오클라우드 Virtual Machine 인스턴스의 상태별 과금에 대한 설명으로 옳은 것은?

해설

카카오클라우드 공식 문서에 따르면, Shelved_offloaded 상태는 인스턴스가 종료되어 하이퍼바이저 상에 할당했던 리소스를 반납한 상태로 과금이 발생하지 않습니다. 반면 Stopped 상태는 인스턴스가 정지되었으나 하이퍼바이저 상에 리소스는 그대로 할당된 상태이므로 과금이 계속 발생합니다.

오답 포인트
  • A: Stopped 상태는 인스턴스가 정지되어 있지만 하이퍼바이저 상에 리소스가 그대로 할당되어 있으므로 과금이 발생합니다. 과금이 발생하지 않는 정지 상태는 Shelved_offloaded입니다.
  • C: Active 상태는 과금이 발생하고, Stopped 상태도 과금이 발생합니다. 두 상태 모두 과금이 발생하지 않는다는 설명은 잘못되었습니다.
  • D: Stopped 상태는 과금이 발생하지만, Shelved_offloaded 상태는 하이퍼바이저 리소스를 반납하여 과금이 발생하지 않습니다. 두 상태의 과금 방식이 다릅니다.
13 q005 Chapter 07 — 복합 난이도 응용

  • C: 데이터베이스와 같이 빠른 쓰기 성능이 필요한 블록 단위 스토리지는 Block Storage(볼륨 서비스)입니다. Object Storage는 대용량 비정형 데이터 저장에 적합합니다.
  • D: NFS 프로토콜을 사용하는 공유 파일 시스템은 File Storage입니다. Object Storage는 파일 공유 환경을 별도로 제공하지 않습니다.

  • B: 보안 그룹은 상태 비관리(Stateless)가 아닌 상태 관리 방식(Stateful)입니다. 따라서 허용된 정책의 응답 트래픽을 별도로 설정할 필요가 없습니다.

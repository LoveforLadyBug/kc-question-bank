## 문제

BCS 인스턴스에 연결하는 블록 스토리지(Block Storage)에 대한 설명으로 옳은 것은?

## 보기

- A. 블록 스토리지는 인스턴스 종료 시 자동으로 삭제되며 데이터 보존이 불가능하다
- B. 블록 스토리지는 인스턴스와 독립적으로 존재하며, 필요에 따라 인스턴스에 연결하거나 분리할 수 있다
- C. 하나의 블록 스토리지 볼륨은 동시에 여러 인스턴스에 자동으로 공유 마운트된다
- D. 블록 스토리지 용량은 최초 설정 이후 절대 변경할 수 없다

## 정답

B

## 해설

BCS의 블록 스토리지는 인스턴스와 독립적인 영구 저장소로, 인스턴스가 종료되더라도 데이터가 보존됩니다. 필요에 따라 인스턴스에 연결(attach)하거나 분리(detach)할 수 있습니다.
이 독립성 덕분에 인스턴스를 교체하거나 삭제할 때도 스토리지에 저장된 데이터를 유지할 수 있어 유연한 데이터 관리가 가능합니다.

## 오답 포인트

- A: 블록 스토리지는 인스턴스 종료 시 자동 삭제되지 않으며, 영구적으로 데이터를 보존합니다.
- C: 일반적으로 블록 스토리지 볼륨은 하나의 인스턴스에 연결되며, 자동 공유 마운트 기능을 기본 제공하지 않습니다.
- D: 블록 스토리지 용량은 필요에 따라 확장 가능하며, 고정 불변이 아닙니다.

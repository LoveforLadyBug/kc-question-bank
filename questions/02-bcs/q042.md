## 문제

BCS 인스턴스 생성 시 운영 체제(OS) 선택에 대한 설명으로 가장 적절한 것은?

## 보기

- A. BCS는 사용자가 직접 OS 이미지를 외부에서 반입하는 것만 허용하며 공식 이미지는 없다
- B. BCS는 Linux 계열과 Windows 계열을 포함한 다양한 공식 OS 이미지를 제공한다
- C. BCS 인스턴스는 한 번 설치된 OS를 변경할 수 없으며, 교체 시 인스턴스를 재생성해야 한다
- D. BCS는 단일 Linux 배포판만 지원하며, 다른 배포판은 사용자가 직접 설치해야 한다

## 정답

B

## 해설

BCS는 Ubuntu, CentOS 등의 Linux 계열과 Microsoft Windows 계열을 포함한 다양한 공식 OS 이미지를 기본 제공합니다.
사용자는 인스턴스 생성 시 워크로드에 맞는 OS를 선택할 수 있으며, 특수 환경을 위한 전용 이미지(예: GPU 드라이버 포함 이미지)도 별도로 제공됩니다.

## 오답 포인트

- A: BCS는 다양한 공식 OS 이미지를 직접 제공하며, 외부 이미지 반입만 허용하는 것이 아닙니다.
- C: 인스턴스 OS 변경 시 재생성이 필요할 수 있지만, 이것이 OS를 '절대 변경할 수 없다'는 의미는 아니며 공식 이미지 제공 여부와 무관합니다.
- D: BCS는 여러 Linux 배포판을 공식 이미지로 지원하며, 단일 배포판만 제공하지 않습니다.

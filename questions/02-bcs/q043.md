## 문제

BCS 가속 컴퓨팅(Accelerated Computing) 인스턴스가 가장 적합한 워크로드는?

## 보기

- A. 소규모 웹 서버 운영 및 경량 API 호출 처리
- B. 딥러닝 모델 학습, 대규모 병렬 수치 연산, 그래픽 렌더링
- C. 관계형 데이터베이스의 트랜잭션 처리 및 ACID 보장
- D. 파일 시스템 백업 및 아카이빙 스토리지 관리

## 정답

B

## 해설

가속 컴퓨팅 인스턴스는 GPU 등의 하드웨어 가속기를 탑재하여 딥러닝 학습, 대규모 병렬 수치 연산, 그래픽 렌더링과 같이 대량의 병렬 연산이 필요한 워크로드에 최적화되어 있습니다.
CPU만으로는 처리 시간이 오래 걸리는 AI/ML 연산이나 HPC 환경에서 GPU의 병렬 처리 능력을 활용함으로써 성능을 크게 향상시킬 수 있습니다.

## 오답 포인트

- A: 소규모 웹 서버나 경량 API 처리는 범용(General Purpose) 인스턴스로 충분하며, 가속 컴퓨팅 인스턴스는 과잉 사양입니다.
- C: 관계형 DB의 트랜잭션 처리는 CPU와 메모리 성능이 중요하므로, 범용 또는 메모리 최적화 인스턴스가 적합합니다.
- D: 파일 백업 및 아카이빙은 스토리지 중심 작업으로, GPU 가속이 필요하지 않습니다.

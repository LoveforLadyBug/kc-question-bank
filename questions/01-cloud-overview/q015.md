## 문제

카카오클라우드에서 VPC 생성 시 IP CIDR 블록 설정에 대한 설명으로 옳은 것은?

## 보기

- A. 공인 IP 대역(예: 203.0.113.0/24)을 포함한 모든 IP 대역에서 자유롭게 CIDR 블록을 지정할 수 있다.
- B. 프라이빗 IP 대역(172.16.0.0 / 192.168.0.0 / 10.0.0.0) 중에서 선택하며, Prefix Length는 16~24 중 설정한다.
- C. 프라이빗 IP 대역에서 선택하되, Prefix Length는 8~16 중 설정하며 서브넷 유형과 무관하게 동일하게 적용된다.
- D. VPC CIDR 블록은 생성 후 언제든지 변경할 수 있으며, 변경 시 기존 서브넷은 자동으로 재구성된다.

## 정답

B

## 해설

카카오클라우드 공식 문서에 따르면, VPC IP CIDR 블록은 프라이빗 IP 대역(172.16.0.0 / 192.168.0.0 / 10.0.0.0) 중에서 사용자가 입력하여 생성할 수 있으며, Prefix Length는 16~24 중에서 선택할 수 있습니다. 또한 Private 유형 서브넷의 경우 Prefix length는 20~26 중 선택하여 상위 VPC보다 좁은 범위로 구성합니다.

## 오답 포인트

- A: VPC CIDR 블록은 공인 IP 대역이 아닌 프라이빗 IP 대역(172.16.0.0, 192.168.0.0, 10.0.0.0)에서만 설정 가능합니다.
- C: Prefix Length 범위는 8~16이 아닌 16~24입니다. 또한 Private 유형 서브넷의 Prefix length(20~26)는 VPC와 다르게 적용됩니다.
- D: VPC CIDR 블록 변경이 가능하다는 내용은 공식 문서에서 언급되지 않습니다. 일반적으로 클라우드 환경에서 VPC CIDR은 생성 후 변경이 불가하며, 기존 서브넷의 자동 재구성도 지원되지 않습니다.

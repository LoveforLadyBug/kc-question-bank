## 문제

카카오클라우드에서 여러 VPC를 성능 저하 없이 유기적으로 연결하고, 온프레미스 네트워크와 중앙 집중식 네트워크를 구축할 때 사용하는 서비스는?

## 보기

- A. VPC Peering
- B. Load Balancer
- C. Transit Gateway
- D. DNS

## 정답

C

## 해설

카카오클라우드 공식 문서에 따르면, Transit Gateway 서비스를 사용하면 성능 저하 없이 여러 VPC를 유기적으로 연결하고, 데이터 센터에서 사용하는 프라이빗 환경과 유사하게 구성할 수 있습니다. 또한 Transit Gateway는 다수의 VPC 및 온프레미스 네트워크를 연결하여 중앙 집중식 네트워크를 구축할 수 있게 해주며, 다중 AZ 환경을 지원하여 전체 시스템의 가용성을 향상하고 빠른 재해복구(DR) 아키텍처를 구현할 수 있습니다.

## 오답 포인트

- A: VPC Peering은 카카오클라우드 공식 문서에서 다수 VPC의 중앙 집중식 연결 수단으로 언급되지 않으며, Transit Gateway가 이 역할을 담당합니다.
- B: Load Balancer는 트래픽을 여러 인스턴스에 분산하는 서비스로, VPC 간 네트워크 연결과는 다른 역할입니다.
- D: DNS는 도메인 이름을 IP 주소로 변환하는 서비스로, VPC 간 네트워크 연결 역할을 하지 않습니다.

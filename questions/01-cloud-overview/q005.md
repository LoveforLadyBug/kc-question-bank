## 문제

카카오클라우드 VPC(Virtual Private Cloud)의 특징으로 옳은 것은?

## 보기

- A. VPC는 동일 프로젝트 내 다른 VPC와 네트워크가 자동으로 연결된다.
- B. VPC는 논리적으로 격리되어 있으며, 다중 AZ를 통해 물리적으로도 분리되어 높은 가용성을 보장한다.
- C. 카카오클라우드에서 BCS 인스턴스를 생성할 때는 VPC 없이도 독립적으로 구성할 수 있다.
- D. Transit Gateway 없이도 여러 VPC를 성능 저하 없이 유기적으로 연결할 수 있다.

## 정답

B

## 해설

카카오클라우드 공식 문서에 따르면, VPC는 논리적으로 격리된 가상 네트워크이며, 다중 AZ(Multi Availability Zone)를 통해 물리적으로도 분리되어 높은 안정성과 가용성을 보장합니다. 이는 카카오클라우드의 핵심 네트워크 설계 원칙입니다.

## 오답 포인트

- A: 카카오클라우드의 VPC는 기본적으로 논리적으로 격리되어 있어 같은 프로젝트 내에서도 VPC 간 자동 연결이 이루어지지 않습니다. VPC 간 연결을 위해서는 Transit Gateway 등의 별도 구성이 필요합니다.
- C: BCS(Beyond Compute Service)의 경우에도 VPC와 같은 네트워크 연결이 반드시 필요합니다. VPC 없이 BCS 인스턴스를 독립적으로 구성하는 것은 불가능합니다.
- D: 여러 VPC를 성능 저하 없이 유기적으로 연결하기 위해서는 Transit Gateway 서비스를 사용해야 합니다. Transit Gateway 없이는 복잡한 네트워크 설정과 성능 저하 문제가 발생할 수 있습니다.

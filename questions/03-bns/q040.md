## 문제

카카오클라우드 Networking 서비스 그룹 내 각 서비스와 그 역할의 연결이 올바른 것은?

## 보기

- A. VPC: 도메인-IP 변환 / Transit Gateway: 트래픽 분산 / Load Balancing: 네트워크 격리
- B. VPC: 네트워크 격리 / Transit Gateway: 다중 VPC 및 온프레미스 연결 / DNS: 도메인-IP 변환
- C. VPC: 트래픽 분산 / Transit Gateway: 도메인 관리 / DNS: 네트워크 격리
- D. Transit Gateway: 네트워크 격리 / Load Balancing: 도메인 관리 / DNS: 트래픽 분산

## 정답

B

## 해설

VPC는 논리적으로 격리된 가상 네트워크 공간을 제공하고, Transit Gateway는 다수의 VPC 및 온프레미스 네트워크를 중앙에서 연결하며, DNS는 도메인을 실제 IP 주소로 변환합니다.
각 서비스는 서로 다른 특화된 네트워크 기술을 담당하고 있으며, 이를 혼동하지 않는 것이 중요합니다.

## 오답 포인트

- A: VPC의 역할은 도메인-IP 변환이 아닌 네트워크 격리이며, 각 서비스의 역할이 전부 뒤바뀌어 있습니다.
- C: VPC는 트래픽 분산이 아닌 네트워크 격리가 주 역할이며, Transit Gateway는 도메인 관리를 담당하지 않습니다.
- D: Transit Gateway는 네트워크 격리가 아닌 다중 VPC 연결이 주 역할이고, DNS의 역할도 트래픽 분산이 아닙니다.

## 문제

카카오클라우드 Networking 서비스 그룹에 포함된 서비스와 그 역할의 연결이 올바른 것은?

## 보기

- A. VPC: 트래픽 분산 / DNS: 네트워크 격리 / Transit Gateway: 도메인 해석
- B. VPC: 논리적 격리 가상 네트워크 / Transit Gateway: 다중 VPC 및 온프레미스 연결 / Load Balancing: 트래픽 분산
- C. Load Balancing: VPC 간 연결 / Transit Gateway: 트래픽 암호화 / DNS: 부하 분산
- D. VPC: 도메인 해석 / Load Balancing: 네트워크 격리 / DNS: 트래픽 분산

## 정답

B

## 해설

카카오클라우드 Networking 서비스 그룹에서 VPC는 논리적으로 격리된 가상 네트워크를 제공하고, Transit Gateway는 다수의 VPC 및 온프레미스 네트워크를 중앙에서 연결하며, Load Balancing은 트래픽을 효율적으로 분산합니다.
각 서비스는 서로 다른 네트워크 계층의 역할을 담당하여 상호 보완적으로 동작합니다.

## 오답 포인트

- A: VPC의 역할은 트래픽 분산이 아닌 네트워크 격리이며, DNS는 도메인 해석을 담당합니다.
- C: VPC 간 연결은 Transit Gateway의 역할이고, DNS는 부하 분산이 아닌 이름 해석 서비스입니다.
- D: 도메인 해석은 DNS의 역할이며, VPC의 역할이 아닙니다.

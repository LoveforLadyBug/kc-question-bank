## 문제

카카오클라우드 네트워킹 서비스 구성에서 각 서비스의 역할을 올바르게 연결한 것은?

## 보기

- A. VPC: 트래픽 분산 / Transit Gateway: 도메인 해석 / Load Balancing: 네트워크 격리
- B. VPC: 네트워크 격리 / Transit Gateway: 다중 VPC 연결 / Load Balancing: 트래픽 분산
- C. VPC: 다중 VPC 연결 / Transit Gateway: 트래픽 분산 / Load Balancing: 도메인 해석
- D. VPC: 도메인 해석 / Transit Gateway: 네트워크 격리 / Load Balancing: 다중 VPC 연결

## 정답

B

## 해설

카카오클라우드에서 VPC는 논리적으로 격리된 네트워크 공간 제공, Transit Gateway는 다수의 VPC와 온프레미스 네트워크를 중앙에서 연결, Load Balancing은 트래픽을 여러 서버에 분산하는 역할을 각각 담당합니다.
세 서비스는 역할이 명확히 구분되며 상호 보완적으로 동작하여 안정적인 클라우드 네트워크 환경을 구성합니다.

## 오답 포인트

- A: 각 서비스의 역할이 모두 뒤바뀐 오답으로, VPC의 핵심은 격리이고 Load Balancing은 분산입니다.
- C: Transit Gateway는 트래픽 분산이 아닌 VPC 간 중앙 연결을 담당하며, VPC가 다중 VPC 연결을 수행하지 않습니다.
- D: 도메인 해석은 DNS 서비스의 역할이며, VPC나 Transit Gateway의 기능이 아닙니다.

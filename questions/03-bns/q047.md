## 문제

카카오클라우드 네트워킹 서비스 그룹에 포함된 서비스와 각 역할의 연결이 올바른 것은?

## 보기

- A. VPC: 트래픽 분산 / DNS: 네트워크 격리 / Transit Gateway: 도메인 변환
- B. VPC: 가상 네트워크 격리 / Load Balancing: 트래픽 분산 / Transit Gateway: 네트워크 허브 연결
- C. Transit Gateway: 트래픽 분산 / VPC: 도메인 변환 / Load Balancing: 네트워크 격리
- D. DNS: 네트워크 허브 연결 / VPC: 트래픽 분산 / Load Balancing: 도메인 변환

## 정답

B

## 해설

카카오클라우드 네트워킹 서비스에서 VPC는 논리적으로 격리된 가상 네트워크 공간을 제공하고, Load Balancing은 트래픽을 효율적으로 분산하며, Transit Gateway는 다수의 VPC와 온프레미스 네트워크를 허브 방식으로 연결하는 역할을 담당합니다.
각 서비스의 역할을 혼동하지 않도록, VPC(격리) → Load Balancing(분산) → Transit Gateway(허브 연결)의 구조로 구분하여 이해하는 것이 중요합니다.

## 오답 포인트

- A: VPC의 역할은 트래픽 분산이 아닌 네트워크 격리이며, DNS는 도메인-IP 변환, Transit Gateway는 네트워크 허브 연결을 담당합니다.
- C: Transit Gateway는 트래픽 분산이 아닌 네트워크 허브 연결을 담당하며, VPC와 Load Balancing의 역할도 서로 뒤바뀌어 있습니다.
- D: DNS는 네트워크 허브 연결이 아닌 도메인-IP 변환 서비스이며, VPC와 Load Balancing의 역할 역시 잘못 기술되어 있습니다.

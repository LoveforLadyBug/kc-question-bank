## 문제

다음 시나리오에서 각 요구사항에 매핑되는 카카오클라우드 Networking 서비스를 올바르게 연결한 것은?

① 여러 서버에 웹 요청을 고르게 나누어 처리 성능을 향상시키고 싶다
② 개발팀과 운영팀의 네트워크를 클라우드에서 논리적으로 분리하고 싶다
③ 여러 부서 VPC와 본사 네트워크를 단일 허브로 묶어 관리하고 싶다

## 보기

- A. ①-DNS / ②-Transit Gateway / ③-VPC
- B. ①-Load Balancing / ②-VPC / ③-Transit Gateway
- C. ①-VPC / ②-Load Balancing / ③-DNS
- D. ①-Transit Gateway / ②-DNS / ③-Load Balancing

## 정답

B

## 해설

① 여러 서버로 트래픽을 분산하는 것은 Load Balancing, ② 네트워크를 논리적으로 격리·분리하는 것은 VPC, ③ 여러 VPC와 온프레미스를 중앙 허브로 연결하는 것은 Transit Gateway의 역할입니다.
이 세 서비스는 카카오클라우드 Networking 그룹의 핵심 구성 요소로, 각각 트래픽 분산·네트워크 격리·광역 연결이라는 서로 다른 문제를 해결합니다.

## 오답 포인트

- A: DNS는 트래픽 분산이 아닌 도메인-IP 변환을 담당하며, Transit Gateway와 VPC의 역할도 잘못 배치되어 있습니다.
- C: VPC는 트래픽 분산이 아닌 네트워크 격리를 위한 서비스이며, Load Balancing과 DNS의 역할도 잘못 매핑되었습니다.
- D: Transit Gateway는 트래픽 부하 분산이 아닌 다중 네트워크 허브 연결을 위한 서비스입니다.

## 문제

카카오클라우드에서 다수의 VPC와 온프레미스 네트워크를 하나의 허브 방식으로 연결하여 중앙 집중식 네트워크를 구성하려 할 때 적합한 서비스는?

## 보기

- A. DNS
- B. Load Balancing
- C. Transit Gateway
- D. Security Group

## 정답

C

## 해설

Transit Gateway는 다수의 VPC와 온프레미스 네트워크를 연결하여 중앙 집중식 허브 네트워크를 구성할 수 있도록 지원하는 서비스입니다.
개별 VPC 간 Peering을 일일이 설정하는 방식과 달리, Transit Gateway를 사용하면 단일 게이트웨이를 통해 여러 네트워크를 효율적으로 관리할 수 있습니다.

## 오답 포인트

- A: DNS는 도메인 이름을 IP 주소로 변환하는 서비스로, 네트워크 간 연결 허브 역할을 하지 않습니다.
- B: Load Balancing은 트래픽을 여러 서버에 분산하는 서비스로, VPC 간 연결 구성과는 무관합니다.
- D: Security Group은 인스턴스 수준의 인바운드/아웃바운드 트래픽을 제어하는 보안 정책이며, 네트워크 연결 허브가 아닙니다.

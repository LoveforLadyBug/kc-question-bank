## 문제

사용자가 웹 브라우저에 "www.example.com"을 입력했을 때 DNS 서비스가 수행하는 동작으로 올바른 것은?

## 보기

- A. 입력된 도메인을 분석하여 가장 응답 속도가 빠른 서버로 트래픽을 전달한다
- B. 도메인 문자열을 해당 사이트를 호스팅하는 서버의 실제 IP 주소로 변환하여 반환한다
- C. 도메인 소유자의 인증서를 검증하고 HTTPS 암호화 세션을 생성한다
- D. 입력된 URL을 파싱하여 경로(Path)별로 서로 다른 백엔드 서버에 라우팅한다

## 정답

B

## 해설

DNS 서비스는 사용자가 입력한 도메인 이름을 해당 웹사이트를 호스팅하는 서버의 실제 IP 주소로 변환(이름 해석)하여 응답하는 역할을 합니다.
이 과정 덕분에 사용자는 복잡한 IP 주소를 외울 필요 없이 기억하기 쉬운 도메인 이름만으로 원하는 서버에 접속할 수 있습니다.

## 오답 포인트

- A: 응답 속도 기반 트래픽 전달은 Load Balancing 또는 글로벌 트래픽 관리 서비스의 기능입니다.
- C: 인증서 검증과 HTTPS 세션 생성은 TLS/SSL 핸드셰이크 과정이며 DNS의 역할이 아닙니다.
- D: URL 경로(Path)별 라우팅은 L7 Load Balancer(Application Load Balancer)의 기능입니다.

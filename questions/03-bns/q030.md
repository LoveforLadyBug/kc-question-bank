## 문제

카카오클라우드 Load Balancing 서비스를 도입했을 때 기대할 수 있는 효과로 가장 적절한 것은?

## 보기

- A. 사용자별로 전용 물리 서버를 할당하여 데이터 유출을 원천 차단한다
- B. 인입 트래픽을 여러 서버에 분산하여 특정 서버의 과부하를 방지한다
- C. VPC 간 네트워크를 중앙 허브로 통합하여 라우팅을 단순화한다
- D. 도메인 질의를 캐싱하여 DNS 응답 속도를 향상시킨다

## 정답

B

## 해설

Load Balancing은 외부로부터 들어오는 트래픽을 복수의 서버로 분산하여, 특정 서버에 부하가 집중되는 것을 방지하고 전체 서비스의 가용성과 성능을 높입니다.
이를 통해 서버 한 대가 다운되더라도 나머지 서버가 트래픽을 처리할 수 있어 서비스 연속성도 확보됩니다.

## 오답 포인트

- A: 전용 물리 서버 할당은 Load Balancing이 아닌 베어메탈 또는 전용 호스팅 서비스 개념입니다.
- C: VPC 간 네트워크 통합 및 라우팅 단순화는 Transit Gateway의 역할입니다.
- D: DNS 응답 캐싱은 DNS Resolver 또는 DNS 캐싱 서비스의 기능으로 Load Balancing과 무관합니다.

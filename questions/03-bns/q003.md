## 문제

카카오클라우드에서 여러 VPC와 온프레미스 네트워크를 중앙에서 연결하여 대규모 네트워크를 구성할 때 사용하는 서비스는?

## 보기

- A. VPC Peering
- B. Transit Gateway
- C. Load Balancing
- D. Security Group

## 정답

B

## 해설

Transit Gateway는 다수의 VPC와 온프레미스 네트워크를 중앙 집중식으로 연결하여, 허브-스포크(Hub-Spoke) 구조의 대규모 네트워크를 구성할 수 있게 해주는 서비스입니다.
다중 AZ(Availability Zone) 환경을 지원하여 고가용성 네트워크 구성이 가능하며, 관리 복잡도를 낮추는 것이 핵심 이점입니다.

## 오답 포인트

- A: VPC Peering은 두 VPC 간 1:1 직접 연결 방식으로, 다수의 VPC를 중앙에서 관리하기에는 적합하지 않습니다.
- C: Load Balancing은 서버 간 트래픽 분산에 특화된 서비스로, VPC 간 네트워크 연결 기능을 제공하지 않습니다.
- D: Security Group은 인스턴스 레벨의 인바운드/아웃바운드 트래픽 제어 규칙으로, 네트워크 연결 서비스가 아닙니다.

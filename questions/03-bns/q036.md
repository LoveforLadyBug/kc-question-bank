## 문제

카카오클라우드 VPC의 다중 AZ(Availability Zone) 지원과 관련하여 올바른 설명은?

## 보기

- A. VPC는 반드시 단일 AZ 내에서만 구성되며 AZ 간 확장이 불가능하다
- B. Transit Gateway 없이는 다중 AZ 네트워크 구성이 전혀 지원되지 않는다
- C. 온프레미스 네트워크와 연결하고 다중 AZ 환경을 지원하는 중앙 집중식 네트워크를 구성할 수 있다
- D. 다중 AZ 지원은 Load Balancing 서비스에서만 단독으로 제공된다

## 정답

C

## 해설

카카오클라우드 VPC는 다수의 VPC와 온프레미스 네트워크를 연결하여 중앙 집중식 네트워크를 구성하고, 다중 AZ 환경을 지원합니다.
이를 통해 특정 가용 영역에 장애가 발생하더라도 다른 AZ를 통해 서비스를 지속할 수 있어 고가용성 아키텍처를 구현할 수 있습니다.

## 오답 포인트

- A: VPC는 다중 AZ 환경을 지원하므로 단일 AZ로만 제한된다는 설명은 잘못되었습니다.
- B: 다중 AZ 구성은 Transit Gateway 없이도 VPC 자체에서 지원 가능합니다.
- D: 다중 AZ 지원은 Load Balancing 단독 기능이 아니며 VPC 레벨에서 제공됩니다.

## 문제

클라우드 운영 시나리오 ①~③과 각각에 가장 적합한 Management 서비스를 올바르게 연결한 것은?

① 월말 감사 보고를 위해 지난 한 달간 발생한 리소스 변경 이력을 추출해야 한다.
② 여러 팀이 사용하는 수백 개의 리소스를 팀 이름과 환경(dev/prod)을 기준으로 분류하고 필터링해야 한다.
③ 특정 서버의 CPU 사용률이 임계값을 넘었을 때 관련 메트릭 데이터를 분석해야 한다.

## 보기

- A. ① Activity Log — ② Dashboard — ③ Service Event
- B. ① Activity Log — ② Resource Manager — ③ Metric
- C. ① Service Event — ② Resource Manager — ③ Activity Log
- D. ① Dashboard — ② Certificate Manager — ③ Metric

## 정답

B

## 해설

① 리소스 변경 이력 추출은 사용자 행동을 자동 기록하는 Activity Log, ② 팀·환경 기준 리소스 분류·필터링은 태그와 메타데이터를 활용하는 Resource Manager, ③ CPU 메트릭 데이터 분석은 실시간 메트릭 수집·저장·분석을 담당하는 Metric이 각각 가장 적합합니다.
각 Management 서비스는 고유한 역할을 가지므로 시나리오별로 목적에 맞는 서비스를 정확히 구분하는 것이 중요합니다.

## 오답 포인트

- A: ②를 Dashboard로 연결했으나, Dashboard는 메트릭·로그 시각화 서비스이며 태그 기반 리소스 분류 기능은 Resource Manager의 역할입니다.
- C: ①을 Service Event로 연결했으나, Service Event는 장애 알림 정책 설정 서비스이며 행동 이력 감사는 Activity Log가 담당합니다.
- D: ②를 Certificate Manager로 연결했으나, Certificate Manager는 SSL 인증서 등록·관리 서비스로 리소스 분류 기능과 무관합니다.

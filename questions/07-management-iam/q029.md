## 문제

DevOps 팀이 카카오클라우드 API를 활용하여 애플리케이션 배포 파이프라인 내에서 특정 엔드포인트의 상태를 자동으로 모니터링하고 이상 감지 시 후속 작업을 자동 실행하려 한다. 이 시나리오에서 API 기반 자동화를 지원하는 Management 서비스의 활용 목적으로 가장 적합한 것은?

## 보기

- A. SSL 인증서의 만료 일정을 달력으로 시각화하고 갱신 알림을 수동 발송한다.
- B. 리소스에 부여된 태그를 기반으로 월별 비용 보고서를 수동으로 생성한다.
- C. API를 통해 애플리케이션 엔드포인트를 모니터링하고 업무 프로세스를 자동화한다.
- D. 사용자 로그인 이력을 수집하여 월간 보안 감사 보고서를 수동으로 작성한다.

## 정답

C

## 해설

카카오클라우드 Management는 API를 활용해 애플리케이션의 엔드포인트를 모니터링하고 업무 프로세스를 자동화할 수 있도록 지원합니다. DevOps 파이프라인에서 API를 통한 상태 확인과 후속 작업 트리거가 이 기능의 핵심 활용 사례입니다.
이는 수동 개입 없이 이상 감지 후 자동 복구, 알림, 배포 중단 등 다양한 자동화 워크플로를 구성할 수 있게 해 줍니다.

## 오답 포인트

- A: 인증서 만료 알림의 수동 발송은 자동화와 반대되는 접근이며, Certificate Manager의 기본 기능 범위에 해당합니다.
- B: 태그 기반 비용 보고서 수동 생성은 Resource Manager와 비용 관리 기능의 조합으로, API 기반 자동화의 목적과 다릅니다.
- D: 로그인 이력 수집은 Activity Log의 역할이며, 수동 보고서 작성은 자동화의 개념과 반대입니다.

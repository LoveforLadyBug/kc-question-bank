## 문제

카카오클라우드 IAM에서 특정 팀원에게 Object Storage 읽기 권한만 허용하고 나머지 리소스 접근은 차단하려 한다. 이때 가장 적합한 접근 방식은?

## 보기

- A. 해당 팀원의 계정을 프로젝트 소유자(Owner)로 설정하여 전체 권한을 부여한 뒤, 불필요한 리소스를 직접 삭제한다.
- B. 해당 팀원에게 Object Storage 읽기 권한만 포함된 역할(Role)을 생성하고 할당한다.
- C. 모든 팀원에게 동일한 전체 권한을 부여한 뒤, 필요에 따라 개별적으로 비밀번호를 공유하여 접근을 제한한다.
- D. Object Storage 리소스를 별도 프로젝트로 이동하여 팀원이 해당 프로젝트에만 접근하도록 네트워크 방화벽을 설정한다.

## 정답

B

## 해설

IAM은 사용자 역할에 따라 세분화된 접근 권한을 부여하여 리소스를 안전하고 효율적으로 관리하는 서비스입니다. 필요한 권한만 포함된 역할을 별도로 생성해 해당 사용자에게 할당하는 것이 최소 권한 원칙(Least Privilege)에 부합하는 올바른 방식입니다.
이를 통해 불필요한 리소스 접근을 원천 차단하면서도, 권한 변경이 필요할 때 역할만 수정하면 되므로 관리 효율성도 높아집니다.

## 오답 포인트

- A: 소유자 권한 전체를 부여하면 최소 권한 원칙에 위배되며, 리소스 삭제로 접근을 제한하는 것은 데이터 손실 위험이 있습니다.
- C: 비밀번호 공유는 보안상 금지된 방식이며, 전체 권한 부여 후 공유로 접근을 통제하는 것은 IAM의 목적과 반대됩니다.
- D: 네트워크 방화벽은 네트워크 레벨의 제어 수단이며, IAM의 역할 기반 접근 제어(RBAC)와는 별개의 메커니즘입니다.

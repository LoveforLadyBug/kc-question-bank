## 문제

카카오클라우드 '서비스 이벤트(Service Event)' 기능의 역할로 가장 적절한 것은?

## 보기

- A. 클라우드 리소스의 메트릭 데이터를 실시간으로 수집하고 장기 보관한다.
- B. 리소스 장애 발생 시 알림을 받을 수 있도록 정책과 수신 채널을 설정하는 서비스이다.
- C. 사용자 로그인 기록과 API 호출 내역을 자동으로 수집하고 감사 로그로 저장한다.
- D. 만료 예정인 SSL 인증서 목록을 조회하고 자동 갱신 스케줄을 등록한다.

## 정답

B

## 해설

서비스 이벤트는 리소스 장애 발생 시 알림을 받을 수 있도록 정책과 수신 채널을 설정하는 서비스로, 운영 중 예기치 않은 장애에 신속하게 대응할 수 있도록 지원합니다.
정책을 통해 어떤 이벤트에 알림을 받을지 정의하고, 수신 채널을 통해 알림을 전달받는 방식으로 동작합니다.

## 오답 포인트

- A: 메트릭 데이터의 실시간 수집 및 장기 저장은 Metric 서비스(모니터링)의 기능입니다.
- C: 사용자 로그인 및 API 호출 내역 수집은 Activity Tracker의 기능에 해당합니다.
- D: SSL 인증서 갱신 스케줄 관리는 인증서 관리(Certificate Manager) 서비스의 기능입니다.

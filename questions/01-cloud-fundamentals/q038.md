## 문제

카카오클라우드에서 데이터베이스를 원하는 시점으로 복구(Point-in-Time Recovery)하기 위해 함께 활용하는 두 가지 기술 요소는 무엇인가요?

## 보기

- A. 스냅샷 이미지와 VPC 피어링 연결
- B. 자동 백업 스케줄과 Read Replica
- C. 스냅샷 백업과 바이너리 로그
- D. Object Storage 버킷과 보안 그룹 정책

## 정답

C

## 해설

카카오클라우드의 데이터베이스는 스냅샷 백업과 바이너리 로그를 함께 활용하여 특정 시점으로 데이터베이스를 복구할 수 있습니다.
스냅샷은 복구의 기준점 역할을 하고, 바이너리 로그는 스냅샷 이후 발생한 변경 사항을 추적하여 원하는 정확한 시점까지 데이터를 재현하는 데 사용됩니다.

## 오답 포인트

- A: VPC 피어링은 네트워크 연결 기능으로 데이터베이스 복구 메커니즘과 무관합니다.
- B: Read Replica는 읽기 부하 분산을 위한 복제본이며, PITR의 핵심 구성 요소가 아닙니다.
- D: Object Storage 버킷과 보안 그룹은 스토리지 및 접근 제어 서비스로, 데이터베이스 시점 복구의 핵심 기술이 아닙니다.

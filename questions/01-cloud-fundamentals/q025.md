## 문제

카카오클라우드에서 데이터베이스를 특정 시점으로 복구할 때 활용하는 두 가지 핵심 구성 요소로 올바른 것은?

## 보기

- A. VPC 피어링 설정과 보안 그룹 규칙
- B. 오브젝트 스토리지 버킷과 쿠버네티스 클러스터
- C. 자동 백업 스케줄과 Metric Export 엔드포인트
- D. 자동 백업(스냅샷)과 바이너리 로그

## 정답

D

## 해설

카카오클라우드 데이터베이스는 자동 백업(스냅샷)과 바이너리 로그를 함께 활용하여 원하는 시점으로 데이터베이스를 복구할 수 있습니다.
스냅샷은 특정 시점의 전체 데이터를 보존하고, 바이너리 로그는 그 이후 발생한 변경 사항을 추적하여 세밀한 시점 복구(PITR)를 가능하게 합니다.

## 오답 포인트

- A: VPC 피어링과 보안 그룹 규칙은 네트워크 연결 및 접근 제어에 사용되며 데이터베이스 복구와 직접적인 관련이 없습니다.
- B: 오브젝트 스토리지 버킷과 쿠버네티스 클러스터는 데이터베이스 시점 복구의 핵심 구성 요소가 아닙니다.
- C: Metric Export 엔드포인트는 모니터링 데이터 외부 전송에 사용되는 기능으로 데이터베이스 복구와 관련이 없습니다.

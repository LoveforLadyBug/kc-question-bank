## 문제

카카오클라우드에서 각 가용 영역에 NAT 인스턴스를 구축하는 주된 목적은 무엇인가요?

## 보기

- A. 퍼블릭 서브넷 인스턴스의 인바운드 트래픽을 차단하기 위해
- B. 프라이빗 서브넷의 인스턴스가 외부 인터넷으로 아웃바운드 통신을 할 수 있도록 하기 위해
- C. 인스턴스 간 블록 스토리지 볼륨을 공유하기 위해
- D. VPC 간 네트워크를 피어링으로 직접 연결하기 위해

## 정답

B

## 해설

NAT 인스턴스는 프라이빗 서브넷에 위치한 인스턴스들이 직접 공인 IP 없이도 외부 인터넷으로 아웃바운드 트래픽을 보낼 수 있도록 네트워크 주소를 변환해주는 역할을 합니다.
카카오클라우드에서는 각 가용 영역에 NAT 인스턴스를 배치하여 가용 영역 장애에도 안정적으로 인터넷 접근을 유지하는 고가용성 네트워크 구성을 권장합니다.

## 오답 포인트

- A: NAT 인스턴스는 인바운드 트래픽 차단이 아니라 프라이빗 서브넷의 아웃바운드 인터넷 접근을 지원하는 역할을 합니다.
- C: 블록 스토리지 공유는 NAT 인스턴스가 아닌 별도의 스토리지 서비스(예: Block Storage)와 관련된 기능입니다.
- D: VPC 간 피어링 연결은 VPC Peering 기능의 역할이며 NAT 인스턴스의 목적과 다릅니다.

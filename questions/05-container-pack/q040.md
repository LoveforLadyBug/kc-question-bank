## 문제

카카오클라우드 Kubernetes Engine을 사용할 때 얻을 수 있는 운영상 이점으로 가장 적절한 것은?

## 보기

- A. 사용자가 Kubernetes 버전 업그레이드 일정을 직접 결정하고 모든 업그레이드 작업을 수동으로 수행해야 한다.
- B. 마스터 노드 인프라를 사용자가 직접 구매·구성해야 하므로 초기 비용이 발생하지 않는다.
- C. 마스터 노드 관리를 플랫폼이 담당하므로 사용자는 애플리케이션과 워크로드 운영에 집중할 수 있다.
- D. 단일 가용 영역 구성만 지원하므로 테스트 환경에 최적화된 서비스이다.

## 정답

C

## 해설

관리형 Kubernetes 서비스의 핵심 이점은 마스터 노드의 프로비저닝·패치·업그레이드·장애 복구 등 복잡한 인프라 운영을 플랫폼이 대신 처리한다는 점입니다.
이를 통해 사용자와 개발팀은 Kubernetes 인프라 유지 관리에 리소스를 낭비하지 않고, 비즈니스 가치를 창출하는 애플리케이션 개발과 워크로드 운영에 집중할 수 있습니다.

## 오답 포인트

- A: 관리형 서비스는 플랫폼이 Kubernetes 버전 관리를 지원하며, 사용자가 모든 업그레이드 작업을 수동으로 수행해야 하는 것이 아닙니다.
- B: 관리형 서비스에서 마스터 노드 인프라는 카카오클라우드가 제공하므로 사용자가 직접 구매·구성하지 않으며, 이는 초기 비용 절감의 이점이지 설명 자체가 사실과 다릅니다.
- D: Kubernetes Engine은 다중 가용 영역(Multi-AZ) 구성을 지원하여 고가용성을 제공하며, 테스트 환경 전용 서비스가 아닙니다.

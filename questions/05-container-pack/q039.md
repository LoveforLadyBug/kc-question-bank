## 문제

카카오클라우드 Container Registry에서 이미지 버전을 체계적으로 관리하기 위한 태그 전략으로 가장 적절한 것은?

## 보기

- A. 태그를 전혀 지정하지 않고 이미지 다이제스트(digest)로만 이미지를 식별하여 관리한다.
- B. 모든 이미지에 `latest` 태그만 단독으로 사용하여 항상 최신 버전을 유지한다.
- C. 시맨틱 버전(v1.2.3) 또는 Git 커밋 해시 등 고유하고 추적 가능한 태그를 이미지에 부여한다.
- D. 이미지 태그는 레지스트리 내부에서 자동 생성되므로 사용자가 별도로 지정할 필요가 없다.

## 정답

C

## 해설

컨테이너 이미지 버전 관리 모범 사례로는 시맨틱 버전(예: v1.2.3)이나 Git 커밋 해시 같은 고유하고 추적 가능한 태그를 활용하는 것이 권장됩니다.
이를 통해 어떤 코드 버전이 배포되었는지 추적하고 문제 발생 시 이전 버전으로 빠르게 롤백하는 것이 가능합니다.

## 오답 포인트

- A: 다이제스트만으로 이미지를 관리하면 사람이 식별하기 어렵고, 배포 파이프라인에서 버전 추적이 매우 불편해집니다.
- B: `latest` 태그만 단독으로 사용하면 실제로 어떤 버전이 배포되었는지 추적이 불가능해지고, 의도치 않은 버전의 이미지가 배포될 위험이 있습니다.
- D: 이미지 태그는 자동으로 생성되지 않으며, 사용자가 docker build 또는 docker tag 명령으로 명시적으로 지정해야 합니다.

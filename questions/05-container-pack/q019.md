## 문제

카카오클라우드 Kubernetes Engine에서 외부 사용자가 클러스터 내 서비스에 접근할 수 있도록 하려면 어떤 구성이 필요한가?

## 보기

- A. 모든 워커 노드에 공인 IP를 직접 할당하고 방화벽을 해제한다.
- B. LoadBalancer 타입의 Service 또는 Ingress를 구성하여 외부 트래픽을 수신한다.
- C. 마스터 노드의 API 서버 포트를 직접 외부에 노출한다.
- D. 클러스터 내 모든 파드에 공인 IP를 부여한다.

## 정답

B

## 해설

Kubernetes에서 외부 트래픽을 클러스터 내부 서비스로 전달하려면 LoadBalancer 타입 Service를 생성하거나 Ingress를 구성하는 것이 표준 방식입니다.
카카오클라우드 환경에서는 LoadBalancer Service 생성 시 클라우드 로드밸런서가 자동으로 프로비저닝되어 외부 접근이 가능해집니다.

## 오답 포인트

- A: 모든 워커 노드에 공인 IP를 할당하고 방화벽을 해제하는 것은 보안상 매우 위험하며 권장되지 않습니다.
- C: 마스터 노드의 API 서버 포트를 직접 외부에 노출하면 클러스터 관리 엔드포인트가 공개되어 심각한 보안 위협이 발생합니다.
- D: 모든 파드에 공인 IP를 부여하는 것은 IP 자원 낭비이며, Kubernetes의 네트워크 설계 원칙에도 맞지 않습니다.

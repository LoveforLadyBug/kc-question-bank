## 문제

카카오클라우드 Kubernetes Engine에서 노드 그룹을 생성할 때 지정할 수 있는 노드 관련 설정으로 가장 적절한 것은?

## 보기

- A. 노드 그룹별로 서로 다른 Kubernetes 버전을 독립적으로 지정할 수 있으며, 마스터 노드 버전과 무관하게 운영된다.
- B. 노드 그룹 생성 시 인스턴스 타입, 노드 수, 가용 영역 등을 설정하여 워커 노드 풀을 구성한다.
- C. 노드 그룹은 오직 하나의 인스턴스 타입만 전체 클러스터에 적용되므로, 그룹별 타입 지정이 불가능하다.
- D. 노드 그룹 생성 후에는 노드 수를 변경할 수 없으며, 재생성을 통해서만 조정이 가능하다.

## 정답

B

## 해설

Kubernetes Engine의 노드 그룹은 워커 노드를 논리적으로 묶는 단위로, 생성 시 인스턴스 타입·노드 수·가용 영역 등의 설정을 지정하여 동일한 사양의 워커 노드 풀을 구성합니다.
노드 그룹 단위로 스케일 업/다운이 가능하며, 서로 다른 사양이 필요한 워크로드는 복수의 노드 그룹을 활용하여 유연하게 대응할 수 있습니다.

## 오답 포인트

- A: 워커 노드의 Kubernetes 버전은 마스터 노드 버전의 범위 내에서 관리되며, 마스터 노드 버전과 완전히 무관하게 독립 운영되지 않습니다.
- C: 노드 그룹별로 서로 다른 인스턴스 타입을 지정할 수 있어, 워크로드 특성에 맞게 다양한 그룹을 구성하는 것이 가능합니다.
- D: 노드 그룹은 생성 이후에도 노드 수 조정(스케일 인/아웃)이 지원되므로, 재생성 없이 운영 중 변경이 가능합니다.

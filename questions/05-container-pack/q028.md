## 문제

Container Registry에서 컨테이너 이미지 태그 전략을 선택할 때, 운영 환경의 안정성과 재현성을 위해 가장 권장되는 방식은?

## 보기

- A. 모든 이미지에 `latest` 태그만 사용하여 항상 최신 이미지를 자동 참조하도록 한다.
- B. 빌드 번호나 Git 커밋 해시 등 고유 식별자를 포함한 불변 태그를 사용한다.
- C. 날짜 기반 태그(예: 20260420)만 사용하여 배포 시점을 쉽게 파악한다.
- D. 태그를 사용하지 않고 이미지 다이제스트(digest)만으로 참조한다.

## 정답

B

## 해설

빌드 번호나 Git 커밋 해시와 같은 고유 식별자를 포함한 불변(immutable) 태그를 사용하면 특정 빌드를 정확히 추적하고 재현할 수 있어 운영 안정성이 높아집니다.
`latest`처럼 가변 태그는 언제 어떤 이미지를 가리키는지 불명확해져 예기치 않은 배포 변경이 발생할 수 있습니다.

## 오답 포인트

- A: `latest` 태그는 덮어쓰기가 가능하여 어떤 버전이 배포됐는지 추적하기 어렵고, 의도치 않은 변경이 운영 환경에 영향을 줄 수 있습니다.
- C: 날짜 기반 태그는 같은 날 여러 빌드가 발생할 경우 구분이 불가능하며, 빌드와 소스 코드 간 연결 추적성이 낮습니다.
- D: 다이제스트만으로 참조하는 방식은 불변성은 보장되지만 가독성이 낮고 실무에서 관리하기 어려워 단독 사용은 권장되지 않습니다.

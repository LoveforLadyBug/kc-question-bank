## 문제

카카오클라우드 Container Registry에서 이미지 태그(tag)를 관리할 때의 모범 사례(Best Practice)로 가장 적절한 것은?

## 보기

- A. 모든 이미지에 `latest` 태그만 사용하여 항상 최신 이미지를 참조하도록 한다.
- B. 빌드 번호나 Git 커밋 해시 등 고유 식별자를 태그로 사용하여 이미지 버전을 추적한다.
- C. 태그 없이 이미지를 푸시하면 레지스트리가 자동으로 최적의 태그를 부여한다.
- D. 태그는 동일 이미지에 하나만 허용되므로 업데이트 시 기존 태그를 항상 삭제해야 한다.

## 정답

B

## 해설

컨테이너 이미지 관리의 모범 사례는 `latest`와 같이 가변적인 태그 대신, 빌드 번호·Git 커밋 해시·시맨틱 버전(예: v1.2.3) 등 고유하고 불변하는 식별자를 태그로 사용하는 것입니다.
이를 통해 특정 버전의 이미지를 정확히 추적하고, 롤백이나 감사(audit) 시 재현 가능한 배포 환경을 보장할 수 있습니다.

## 오답 포인트

- A: `latest` 태그는 가변적이어서 어떤 이미지가 실제로 배포되는지 추적하기 어렵고, 예기치 않은 변경이 발생할 수 있습니다.
- C: 태그 없이 이미지를 푸시하면 자동으로 `latest` 태그가 부여되며, 레지스트리가 최적 태그를 선정하는 기능은 없습니다.
- D: 하나의 이미지에 여러 태그를 동시에 부여하는 것이 가능하며, 태그 삭제가 강제되지 않습니다.

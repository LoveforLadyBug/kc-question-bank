## 문제

Container Registry에 저장된 이미지를 Kubernetes Engine 클러스터에서 사용하기 위해 가장 먼저 수행해야 하는 작업은?

## 보기

- A. 이미지를 tar 파일로 내보내 워커 노드에 직접 복사한다.
- B. 레지스트리 인증 정보를 포함한 Secret을 클러스터에 등록하거나 권한을 연동한다.
- C. 워커 노드에 Docker Hub 계정을 별도로 생성한다.
- D. 이미지를 반드시 퍼블릭으로 전환한 뒤 클러스터에서 pull한다.

## 정답

B

## 해설

프라이빗 Container Registry의 이미지를 Kubernetes에서 사용하려면 레지스트리 인증 정보(imagePullSecrets 또는 서비스 계정 연동)를 클러스터에 등록해야 이미지를 정상적으로 pull할 수 있습니다.
카카오클라우드 환경에서는 IAM 기반 권한 연동 또는 Secret 방식으로 인증을 처리합니다.

## 오답 포인트

- A: tar 복사 방식은 자동화·확장성이 없으며 Kubernetes의 표준 이미지 관리 방식이 아닙니다.
- C: Docker Hub 계정은 카카오클라우드 Container Registry 인증과 무관합니다.
- D: 이미지를 퍼블릭으로 전환하면 보안 위협이 발생하므로 권장되지 않으며, 프라이빗 상태에서 인증을 통해 사용하는 것이 올바른 방법입니다.

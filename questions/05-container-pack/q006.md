## 문제

개발자가 로컬에서 빌드한 도커 이미지를 카카오클라우드 Container Registry에 업로드하려고 할 때, 올바른 작업 순서는?

## 보기

- A. docker build → docker push → docker login → 이미지 태그 지정
- B. docker login → 이미지 태그 지정 → docker push
- C. 이미지 태그 지정 → docker push → docker login → docker build
- D. docker login → docker build → docker push → 이미지 태그 지정

## 정답

B

## 해설

Container Registry에 이미지를 업로드하려면 먼저 `docker login` 명령으로 레지스트리에 인증한 후, 업로드할 이미지에 레지스트리 주소를 포함한 태그를 지정하고, 마지막으로 `docker push` 명령으로 이미지를 업로드합니다.
이미지가 이미 로컬에 빌드된 상태라면 별도의 `docker build` 단계는 필요하지 않으며, 인증 → 태그 지정 → 푸시 순서가 핵심입니다.

## 오답 포인트

- A: docker login 없이 push를 먼저 시도하면 인증 오류가 발생하며, 순서가 올바르지 않습니다.
- C: docker login 전에 push를 시도하는 순서로, 인증이 선행되지 않아 실패합니다.
- D: 이미지 빌드 후 태그 지정 없이 push하면 레지스트리 경로를 알 수 없어 업로드가 실패합니다.

## 문제

카카오클라우드 Kubernetes Engine이 관리형(Managed) 서비스로 제공될 때 사용자가 직접 관리하지 않아도 되는 구성 요소는?

## 보기

- A. 워커 노드에 배포된 애플리케이션 파드(Pod)
- B. 사용자가 정의한 ConfigMap과 Secret
- C. Kubernetes 컨트롤 플레인(Control Plane, 마스터 노드)
- D. 사용자 정의 네임스페이스(Namespace)

## 정답

C

## 해설

관리형 Kubernetes 서비스에서는 API 서버, etcd, 스케줄러, 컨트롤러 매니저 등으로 구성된 컨트롤 플레인(마스터 노드)을 클라우드 공급자(카카오클라우드)가 직접 운영·관리합니다.
사용자는 컨트롤 플레인의 설치, 업그레이드, 장애 복구에 개입할 필요 없이 워크로드 운영에만 집중할 수 있습니다.

## 오답 포인트

- A: 파드는 사용자가 직접 배포하고 관리하는 워크로드 단위입니다.
- B: ConfigMap과 Secret은 사용자가 애플리케이션 설정을 위해 직접 생성·관리합니다.
- D: 네임스페이스는 사용자가 논리적 격리 목적으로 직접 생성하고 관리하는 리소스입니다.

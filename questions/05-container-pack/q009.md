## 문제

카카오클라우드 Kubernetes Engine에서 클러스터의 고가용성(HA)을 확보하기 위한 구성으로 가장 적절한 것은?

## 보기

- A. 모든 워커 노드를 단일 가용 영역에 집중 배치하여 네트워크 지연을 최소화한다.
- B. 노드 그룹을 여러 가용 영역에 걸쳐 분산 배치하여 가용 영역 장애에 대비한다.
- C. 컨트롤 플레인 노드를 사용자가 직접 추가하여 이중화 구성을 적용한다.
- D. 단일 대형 노드에 모든 파드를 배치하여 관리 복잡도를 줄인다.

## 정답

B

## 해설

고가용성을 확보하려면 워커 노드를 여러 가용 영역(Availability Zone)에 분산 배치하여 특정 가용 영역에 장애가 발생하더라도 서비스가 지속될 수 있도록 구성해야 합니다.
노드 그룹 단위로 가용 영역을 지정할 수 있으며, 이는 Kubernetes의 파드 스케줄링과 결합하여 실질적인 장애 허용성(fault tolerance)을 제공합니다.

## 오답 포인트

- A: 단일 가용 영역 집중 배치는 해당 영역 장애 시 전체 서비스 중단으로 이어지므로 고가용성에 반하는 구성입니다.
- C: 관리형 서비스에서는 컨트롤 플레인을 사용자가 직접 추가하거나 구성할 수 없으며, 카카오클라우드가 관리합니다.
- D: 단일 노드 집중 구성은 해당 노드 장애 시 모든 서비스가 중단되므로 고가용성 목적에 적합하지 않습니다.

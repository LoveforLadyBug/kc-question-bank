## 문제

카카오클라우드 Kubernetes Engine에서 클러스터를 생성하기 위해 반드시 사전에 구성해야 하는 네트워크 인프라로 가장 적절한 것은?

## 보기

- A. 전용 물리 회선(Direct Connect)과 온프레미스 데이터센터 연결
- B. VPC(Virtual Private Cloud)와 서브넷(Subnet)
- C. 글로벌 CDN 엔드포인트와 DNS Zone
- D. 퍼블릭 IP 대역과 인터넷 게이트웨이만으로 구성된 플랫 네트워크

## 정답

B

## 해설

카카오클라우드 Kubernetes Engine은 VPC 기반으로 동작하므로, 클러스터 생성 전에 VPC와 그 안에 서브넷이 반드시 구성되어 있어야 합니다.
클러스터와 노드 그룹은 지정된 VPC와 서브넷 내에 배치되어 네트워크 격리 및 보안 정책을 적용받습니다.

## 오답 포인트

- A: 전용 물리 회선 연결은 온프레미스 연동 시 선택적으로 사용하는 구성이며, Kubernetes Engine 클러스터 생성의 필수 전제 조건이 아닙니다.
- C: CDN 엔드포인트나 DNS Zone은 서비스 노출 단계에서 선택적으로 구성하는 요소이며, 클러스터 생성의 사전 필수 요건이 아닙니다.
- D: 퍼블릭 IP 대역만으로 이루어진 플랫 네트워크는 카카오클라우드 Kubernetes Engine의 네트워크 구성 방식과 맞지 않으며, VPC 기반 격리 구조가 전제됩니다.

## 문제

운영 중인 Kubernetes Engine 클러스터에서 특정 워크로드의 트래픽이 급증했을 때, 가장 적절한 노드 관리 대응 방안은?

## 보기

- A. 기존 클러스터를 삭제하고 더 큰 인스턴스 타입으로 새 클러스터를 생성한다.
- B. 마스터 노드의 CPU를 사용자가 직접 업그레이드한다.
- C. 해당 워크로드용 노드 그룹의 노드 수를 늘려 수평 확장한다.
- D. 다른 클러스터의 노드를 현재 클러스터로 이동시킨다.

## 정답

C

## 해설

Kubernetes Engine에서는 노드 그룹 단위로 노드 수를 조정할 수 있어, 트래픽 급증 시 해당 워크로드가 속한 노드 그룹의 노드 수를 늘리는 수평 확장이 가장 적절합니다.
이 방식은 클러스터 중단 없이 용량을 동적으로 증가시킬 수 있어 운영 연속성을 유지합니다.

## 오답 포인트

- A: 클러스터를 삭제하고 재생성하면 서비스 중단이 발생하므로 운영 중인 환경에서 적절하지 않습니다.
- B: 마스터 노드는 카카오클라우드가 관리하며 사용자가 직접 하드웨어를 업그레이드할 수 없습니다.
- D: 클러스터 간 노드 이동은 지원되지 않는 구성이며, 표준적인 Kubernetes 운영 방식이 아닙니다.

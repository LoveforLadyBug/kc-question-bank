## 문제

카카오클라우드 Kubernetes Engine이 '관리형(Managed)' 서비스로 분류되는 가장 핵심적인 이유는?

## 보기

- A. 사용자가 워커 노드의 운영체제를 직접 선택하고 패치를 수동으로 적용해야 한다.
- B. 컨트롤 플레인(마스터 노드) 운영과 관리를 카카오클라우드가 대신 담당한다.
- C. 클러스터 내 모든 파드의 배포 전략을 카카오클라우드가 자동으로 결정한다.
- D. 사용자가 정의한 YAML 매니페스트 없이도 애플리케이션이 자동으로 실행된다.

## 정답

B

## 해설

관리형 Kubernetes 서비스의 핵심은 컨트롤 플레인(마스터 노드) 구성 및 유지보수를 클라우드 공급자가 담당하는 것입니다.
카카오클라우드 Kubernetes Engine은 사용자가 마스터 노드를 직접 설치·운영할 필요 없이 VPC 기반에서 쉽게 Kubernetes를 실행할 수 있도록 지원합니다.

## 오답 포인트

- A: 관리형 서비스에서도 워커 노드의 일부 관리는 사용자 책임이 있을 수 있으나, 이것이 '관리형'으로 불리는 핵심 이유가 아닙니다. 핵심은 컨트롤 플레인 관리입니다.
- C: 파드의 배포 전략(롤링 업데이트, 블루-그린 등)은 여전히 사용자가 정의하며, 카카오클라우드가 자동으로 결정하지 않습니다.
- D: 관리형 서비스라도 애플리케이션 실행을 위해 사용자는 YAML 매니페스트 등을 통해 원하는 상태를 정의해야 합니다.

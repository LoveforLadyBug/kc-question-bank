## 문제

Kubernetes Engine에서 특정 워크로드(예: GPU 연산)를 전담하는 전용 노드 그룹을 구성할 때, 해당 노드에만 특정 파드가 스케줄링되도록 제어하는 Kubernetes 기능 조합으로 가장 적절한 것은?

## 보기

- A. ConfigMap과 Secret
- B. PersistentVolume과 StorageClass
- C. Taint/Toleration과 NodeSelector(또는 NodeAffinity)
- D. HorizontalPodAutoscaler와 PodDisruptionBudget

## 정답

C

## 해설

특정 노드에만 특정 파드가 스케줄링되도록 제어하려면 노드에 Taint를 설정하여 일반 파드의 배치를 막고, 해당 파드에 Toleration을 부여하여 해당 노드에 배치될 수 있게 합니다. 추가로 NodeSelector 또는 NodeAffinity를 통해 특정 노드 그룹을 명시적으로 지정할 수 있습니다.
이 조합은 GPU 노드, 고메모리 노드 등 특수 목적 노드 그룹을 운영할 때 표준적으로 사용되는 패턴입니다.

## 오답 포인트

- A: ConfigMap과 Secret은 애플리케이션 설정 및 민감 정보를 주입하는 용도로, 스케줄링 제어와는 관련이 없습니다.
- B: PersistentVolume과 StorageClass는 스토리지 프로비저닝과 관련된 리소스로, 파드의 노드 배치를 제어하지 않습니다.
- D: HorizontalPodAutoscaler는 파드 수를 자동 조정하고, PodDisruptionBudget은 유지보수 중 파드 가용성을 보장하는 기능으로, 특정 노드 지정 스케줄링과 다릅니다.

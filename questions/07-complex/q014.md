## 문제

카카오클라우드 API 인증 토큰의 **권한이 변경되는 경우**에 대한 설명으로 옳은 것은?

## 보기

- A. 프로젝트 역할이 변경되어도 기존 토큰은 만료 시점까지 원래 권한을 유지한다.
- B. 프로젝트 역할이 변경되면 토큰 발급 시점과 현재 역할을 비교하여 일치하는 역할 또는 권한만 상속된다.
- C. 프로젝트 역할이 축소되면 기존 토큰은 즉시 무효화되고 새 토큰을 발급받아야 한다.
- D. 프로젝트 역할이 변경되면 관리자가 수동으로 기존 토큰을 폐기해야 한다.

## 정답

B

## 해설

카카오클라우드 공식 문서에 따르면, 소속 프로젝트 역할이 변경된 경우 토큰은 즉시 무효화되지 않고 **토큰 발급 시점과 현재 역할을 비교하여 일치하는 역할 또는 권한만 상속**됩니다. 예를 들어, 프로젝트 관리자에서 프로젝트 멤버로 역할이 낮아지면 해당 토큰은 프로젝트 멤버 권한만 상속받습니다. 이를 통해 역할 변경 후에도 토큰 재발급 없이 변경된 권한으로 서비스를 이용할 수 있습니다.

## 오답 포인트

- A: 원래 권한을 그대로 유지하는 것이 아니라, 현재 역할에 맞는 권한으로 자동 조정됩니다.
- C: 역할 변경 시 토큰이 즉시 무효화되지는 않으며, 변경된 권한으로 상속됩니다. 단, 역할이 완전히 삭제(프로젝트에서 내보내기)된 경우에는 토큰이 만료됩니다.
- D: 수동 폐기 없이 자동으로 권한이 조정됩니다.

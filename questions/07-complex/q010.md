## 문제

카카오클라우드 API로 **사용자가 속한 프로젝트 목록을 조회**할 때, Request Header에 포함해야 하는 항목으로 옳은 것은?

## 보기

- A. `Content-Type: application/json`
- B. `X-Auth-Token: {tokenId}`
- C. `Authorization: Bearer {tokenId}`
- D. `X-Subject-Token: {tokenId}`

## 정답

B

## 해설

카카오클라우드 공식 문서에 따르면, 프로젝트 목록 조회 API(`GET https://iam.kakaocloud.com/identity/v3/users/{userId}/projects`)를 호출할 때 Request Header에 **`X-Auth-Token: {tokenId}`**을 필수로 포함해야 합니다. `{tokenId}`는 발급받은 API 인증 토큰 값을 사용합니다.

## 오답 포인트

- A: `Content-Type: application/json`은 API 인증 토큰 발급 요청 시 필요한 헤더입니다. 프로젝트 목록 조회 시에는 `X-Auth-Token`을 사용합니다.
- C: 카카오클라우드 API에서는 `Authorization: Bearer` 방식이 아닌 `X-Auth-Token` 헤더를 사용합니다.
- D: `X-Subject-Token`은 토큰 발급 응답(Response Header)에서 발급된 토큰 값을 확인하는 항목입니다. 요청(Request Header)에서 인증에 사용하는 항목은 `X-Auth-Token`입니다.

## 문제

카카오클라우드 API 인증 토큰 발급 후, 해당 사용자의 **프로젝트 역할이 관리자(Admin)에서 멤버(Member)로 변경**된 경우 발생하는 동작으로 옳은 것은?

## 보기

- A. 기존 토큰은 즉시 만료되며 재발급해야 한다.
- B. 기존 토큰은 유효 기간(12시간)까지 관리자 권한을 유지한다.
- C. 기존 토큰은 변경된 멤버(Member) 권한을 상속받아 계속 유효하다.
- D. 역할 변경은 토큰의 권한에 영향을 미치지 않는다.

## 정답

C

## 해설

카카오클라우드 공식 문서에 따르면, 소속 프로젝트 역할이 변경된 경우, 기존 토큰은 즉시 만료되지 않고 **토큰 발급 시점의 역할과 현재 역할을 비교하여 일치하는 역할 또는 권한만 상속**받습니다. 따라서 관리자(Admin)에서 멤버(Member)로 변경된 경우, 기존 토큰은 멤버 권한으로 계속 유효합니다.

## 오답 포인트

- A: 역할 변경은 토큰을 즉시 만료시키지 않습니다. 토큰의 즉시 만료(권한 만료)는 프로젝트에서 내보내기(역할 삭제)되거나, IAM 액세스 키를 직접 삭제한 경우에 발생합니다.
- B: 역할이 낮아진 경우 기존의 높은 권한(관리자)을 유지하지 않습니다. 변경된 낮은 권한(멤버)을 상속받습니다.
- D: 역할 변경은 토큰의 권한에 영향을 미칩니다. 변경된 역할에 따라 토큰 권한이 조정됩니다.

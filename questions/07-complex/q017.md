## 문제

특정 서버에서 카카오클라우드 **API 인증 토큰 발급 시 Timeout**이 반복 발생하고 있다. 공식 문서에서 제시하는 원인과 해결 방법으로 옳은 것은?

## 보기

- A. API 엔드포인트 URL이 변경되었을 가능성이 높으므로 최신 URL로 교체한다.
- B. IAM 액세스 키가 만료되었을 가능성이 높으므로 새 액세스 키를 발급받는다.
- C. 해당 서버 IP에서 루프 코드로 인한 반복 요청으로 IP가 차단되었을 수 있으며, 새 IP 할당 또는 카카오클라우드 문의를 통해 차단 해제를 요청할 수 있다.
- D. Content-Type 헤더 누락이 원인이므로 `application/json`을 추가하면 해결된다.

## 정답

C

## 해설

카카오클라우드 공식 문서에 따르면, 특정 서버에서 API 인증 토큰 발급 시 Timeout이 반복되는 원인은 **해당 서버 IP에서 루프 코드로 인한 반복 토큰 발급 요청**으로 인해 해당 IP가 차단되었기 때문일 수 있습니다. 해결 방법으로는 같은 IP의 다른 서버에서 토큰 발급 가능 여부를 확인한 후, 새로운 IP 주소를 할당한 서버를 운영하거나, 카카오클라우드 문의를 통해 해당 IP 차단 해제를 요청하는 방법이 있습니다.

## 오답 포인트

- A: Timeout 문제는 URL 변경이 아닌 IP 차단이 주요 원인입니다.
- B: IAM 액세스 키 만료 시에는 Timeout이 아닌 인증 실패 에러가 발생합니다.
- D: Content-Type 헤더 누락 시에는 인증 토큰 발급이 거부되는 에러가 발생하며, Timeout과는 다른 문제입니다.

## 문제

클라이언트 애플리케이션에서 카카오클라우드 **API 인증 토큰 발급 API를 직접 호출할 때 CORS 에러**가 발생하는 경우, 공식 문서에서 권장하는 해결 방법은?

## 보기

- A. Request Header에 `Origin: *` 를 추가하면 CORS 에러가 해결된다.
- B. IAM 콘솔에서 특정 도메인을 허용 목록(whitelist)에 등록한다.
- C. 클라이언트 요청을 받아 IAM 서버로 전달하는 Proxy 서버를 설정한다.
- D. API 인증 토큰 대신 IAM 액세스 키를 직접 사용한다.

## 정답

C

## 해설

카카오클라우드 공식 문서에 따르면, 클라이언트 애플리케이션에서 IAM API 엔드포인트로 직접 토큰 발급 요청을 보낼 경우 CORS(Cross-Origin Resource Sharing) 에러가 발생할 수 있습니다. 이는 카카오클라우드 IAM API가 보안상 외부 출처(origin)로부터의 요청을 허용하지 않기 때문입니다. 공식 권장 해결 방법은 **Proxy 서버를 설정하여 클라이언트 요청을 받아 IAM 서버로 전달**하는 방식입니다.

## 오답 포인트

- A: 카카오클라우드 IAM API는 외부 출처 요청 자체를 허용하지 않으므로, 클라이언트 측 헤더 추가로는 해결되지 않습니다.
- B: IAM 콘솔에서 허용 도메인 화이트리스트 등록 기능은 제공하지 않습니다.
- D: IAM 액세스 키 자체도 클라이언트에서 직접 사용 시 동일한 CORS 문제가 발생할 수 있습니다.

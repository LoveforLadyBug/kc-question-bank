## 문제

신규 프로젝트에서 사용자 정의 데이터 타입과 복잡한 객체 상속 구조를 데이터베이스 레벨에서 지원해야 하는 요구사항이 있을 때, 카카오클라우드 Data Store에서 가장 적합한 서비스는?

## 보기

- A. MySQL — RDBMS 기반으로 높은 호환성을 제공하기 때문에
- B. MemStore — 인메모리 방식으로 빠른 처리 속도를 제공하기 때문에
- C. PostgreSQL — ORDBMS 기반으로 객체 지향 기능을 지원하기 때문에
- D. MySQL — 오픈 소스 기반으로 확장성이 가장 뛰어나기 때문에

## 정답

C

## 해설

PostgreSQL은 ORDBMS(객체-관계형 데이터베이스 관리 시스템)로, 사용자 정의 데이터 타입, 함수, 연산자, 상속 등 객체 지향적 기능을 데이터베이스 레벨에서 기본 지원합니다.
복잡한 데이터 구조와 확장 가능한 타입 시스템이 필요한 프로젝트에는 순수 RDBMS인 MySQL보다 PostgreSQL이 더 적합합니다.

## 오답 포인트

- A: MySQL은 RDBMS로 사용자 정의 타입·객체 상속 등 ORDBMS 고유 기능을 기본적으로 지원하지 않습니다.
- B: MemStore는 인메모리 데이터 스토어로 객체 상속이나 복잡한 스키마 구조를 지원하지 않습니다.
- D: MySQL의 오픈 소스 기반 확장성은 해당 요구사항(객체 상속, 사용자 정의 타입)과 직접적인 관련이 없습니다.

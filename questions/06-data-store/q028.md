## 문제

카카오클라우드 MemStore가 MySQL이나 PostgreSQL보다 유리한 상황으로 가장 적절한 것은?

## 보기

- A. 복잡한 외래 키 관계와 다중 테이블 JOIN이 빈번한 금융 거래 시스템
- B. 데이터 일관성(ACID)이 최우선인 재고 관리 시스템
- C. 응답 속도가 매우 중요한 실시간 랭킹 또는 세션 캐시 시스템
- D. 장기 보관이 필요한 정형 데이터를 스키마 기반으로 저장하는 ERP 시스템

## 정답

C

## 해설

MemStore는 인메모리 방식으로 데이터를 처리하여 디스크 I/O 없이 극히 빠른 읽기·쓰기를 제공하므로, 실시간 랭킹·세션 캐시처럼 응답 속도가 핵심인 사용 사례에 적합합니다.
반면 MySQL·PostgreSQL은 디스크 기반 영구 저장과 ACID 트랜잭션에 강점이 있어 데이터 정합성이 중요한 트랜잭션 처리 시스템에 적합합니다.

## 오답 포인트

- A: 다중 테이블 JOIN과 외래 키 관계가 복잡한 시스템은 RDBMS(MySQL·PostgreSQL)가 적합합니다.
- B: ACID 일관성이 최우선인 재고 관리 시스템은 디스크 기반 RDBMS가 더 안정적입니다.
- D: 장기 보관이 필요한 정형 데이터는 디스크 기반 RDBMS에 적합하며, 인메모리 MemStore는 주 저장소보다 캐시 계층으로 활용됩니다.

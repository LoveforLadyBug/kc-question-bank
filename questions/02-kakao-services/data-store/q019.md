## 문제

카카오클라우드 **MySQL** 서비스에서 **데이터베이스 접근 보안**을 강화하는 방법으로 옳은 것은?

## 보기

- A. 모든 클라이언트의 접근을 기본적으로 허용하고, 차단이 필요한 IP만 블랙리스트에 등록한다.
- B. 보안 그룹을 연결하여 특정 IP 대역의 클라이언트만 데이터베이스에 접근할 수 있도록 허용한다.
- C. 데이터베이스 포트를 주기적으로 변경하여 보안을 유지한다.
- D. 퍼블릭 IP를 통해 외부에서 직접 접근하도록 강제한다.

## 정답

B

## 해설

카카오클라우드 공식 문서에 따르면, MySQL 인스턴스에 **보안 그룹을 연결**하여 데이터베이스 보안을 강화하고, **특정 IP 대역의 클라이언트만 데이터베이스에 접근할 수 있도록 허용**할 수 있습니다. 또한 데이터베이스가 설치된 VM은 논리적으로 격리된 VPC 네트워크에 배포되어 허용되지 않은 외부 접근을 차단합니다.

## 오답 포인트

- A: 기본 모든 허용 후 블랙리스트 방식은 보안 그룹의 화이트리스트 방식과 반대입니다.
- C: 포트를 주기적으로 변경하는 것은 공식 문서에 명시된 보안 방법이 아닙니다.
- D: MySQL은 VPC 내 프라이빗 IP를 통한 안전한 통신을 지원합니다. 퍼블릭 IP 강제는 보안상 좋지 않습니다.

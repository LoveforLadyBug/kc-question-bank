## 문제

카카오클라우드 MemStore에서 **고가용성(HA)** 기능을 사용하는 경우, Primary 노드 장애 발생 시 동작으로 옳은 것은?

## 보기

- A. 사용자가 수동으로 Replica 노드를 Primary로 승격(Promote)해야 한다.
- B. 자동으로 가장 최신 데이터를 보유한 Replica 노드가 Primary로 승격되고, 새로운 Replica 노드가 자동 생성된다.
- C. 클러스터 전체가 재시작되며, 모든 데이터가 초기화된다.
- D. Primary 노드 장애 시 고가용성 기능이 작동하지 않으며, 사용자가 직접 새 클러스터를 생성해야 한다.

## 정답

B

## 해설

카카오클라우드 공식 문서에 따르면, MemStore에서 고가용성 기능을 사용하는 경우 Primary 노드 장애 시 자동으로 가장 최신 데이터를 보유한 Replica 노드를 선택하여 Primary로 승격(Automatic Failover)하며, 새로운 Replica 노드를 자동 생성하여 Replica 개수를 유지(Auto Healing)합니다. Replica 노드 장애 시에도 자동으로 새 노드로 대체하여 Replica 개수를 유지합니다.

## 오답 포인트

- A: 고가용성 기능을 사용하는 경우에는 사용자가 수동으로 승격할 수 없습니다. 승격은 서비스에서 필요하다고 판단할 때 자동으로 수행됩니다. 수동 승격은 고가용성 기능을 사용하지 않는 경우에만 가능합니다.
- C: 고가용성 장애 발생 시 클러스터 전체가 재시작되거나 데이터가 초기화되지 않습니다. Automatic Failover를 통해 서비스 연속성이 유지됩니다.
- D: 고가용성 기능은 Primary 노드 장애 시 자동으로 동작하며, 사용자가 새 클러스터를 생성할 필요가 없습니다.

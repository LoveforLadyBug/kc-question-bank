## 문제

카카오클라우드 Object Storage에서 **스토리지 관리자(storage.admin)**와 **스토리지 편집자(storage.editor)**의 차이점으로 옳은 것은?

## 보기

- A. 스토리지 관리자는 버킷 생성 권한을 가지고, 스토리지 편집자는 버킷 생성 권한이 없다.
- B. 스토리지 관리자는 버킷의 권한 정책(getIamPolicy/setIamPolicy)을 관리할 수 있고, 스토리지 편집자는 권한 정책 관리 권한이 없다.
- C. 스토리지 편집자는 객체를 삭제할 수 있지만, 스토리지 관리자는 객체 삭제 권한이 없다.
- D. 스토리지 관리자와 스토리지 편집자의 권한은 동일하며, 역할 이름만 다르다.

## 정답

B

## 해설

카카오클라우드 공식 문서에 따르면, 스토리지 관리자(storage.admin)는 버킷과 객체를 관리할 수 있는 전체 권한을 가지며 버킷 권한 정책 조회(getIamPolicy)와 설정(setIamPolicy)을 포함합니다. 반면 스토리지 편집자(storage.editor)는 버킷의 권한 정책 관리 권한(getIamPolicy/setIamPolicy)이 없으며, 버킷과 객체의 일반적인 생성·수정·삭제만 가능합니다.

## 오답 포인트

- A: 버킷 생성 권한은 IAM 역할(프로젝트 관리자, 프로젝트 멤버, Object Storage 매니저)에 의해 제어됩니다. 스토리지 관리자(storage.admin)와 스토리지 편집자(storage.editor)는 모두 버킷 생성 후 버킷 내 리소스 관리 역할이며, 두 역할 모두 storage.buckets.create 권한을 포함하지 않습니다.
- C: 스토리지 관리자는 객체 삭제(storage.objects.delete) 권한을 포함하며, 스토리지 편집자도 객체 삭제 권한을 가집니다. 두 역할 모두 객체 삭제가 가능합니다.
- D: 스토리지 관리자와 스토리지 편집자의 권한은 다릅니다. 가장 큰 차이는 권한 정책(IamPolicy) 관리 여부입니다.

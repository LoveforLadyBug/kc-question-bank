## 문제

카카오클라우드 Kubernetes Engine에서 지원하는 CNI 플러그인과 관련된 설명으로 옳은 것은?

## 보기

- A. Flannel과 Weave Net 두 가지 CNI 플러그인을 지원하며, 클러스터 생성 후에도 변경 가능하다.
- B. Calico와 Cilium 두 가지 CNI 플러그인을 지원하며, 클러스터 생성 후에도 자유롭게 변경할 수 있다.
- C. Calico와 Cilium 두 가지 CNI 플러그인을 지원하며, 클러스터 생성 시 선택한 CNI는 이후 변경할 수 없다.
- D. CNI 플러그인은 카카오클라우드가 자동으로 선택하며, 사용자가 직접 선택할 수 없다.

## 정답

C

## 해설

카카오클라우드 공식 문서에 따르면, Kubernetes Engine은 Calico와 Cilium 두 가지 CNI(Container Network Interface) 플러그인을 지원합니다. 클러스터 생성 시 CNI 플러그인을 선택할 수 있지만, 한번 선택한 CNI 플러그인은 클러스터 생성 이후 변경할 수 없습니다. 따라서 클러스터 생성 전 적절한 CNI를 신중하게 선택해야 합니다.

## 오답 포인트

- A: Kubernetes Engine이 지원하는 CNI 플러그인은 Flannel과 Weave Net이 아니라 Calico와 Cilium입니다.
- B: CNI 플러그인 선택지(Calico, Cilium)는 맞지만, 클러스터 생성 후에도 자유롭게 변경할 수 있다는 설명이 잘못되었습니다. CNI는 생성 후 변경이 불가능합니다.
- D: CNI 플러그인은 클러스터 생성 시 사용자가 직접 선택합니다. 카카오클라우드가 자동으로 선택한다는 설명은 잘못되었습니다.

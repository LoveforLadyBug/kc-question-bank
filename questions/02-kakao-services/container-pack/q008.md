## 문제

카카오클라우드 Kubernetes Engine에서 클러스터를 삭제할 때 **자동으로 함께 삭제**되는 리소스를 모두 올바르게 나열한 것은?

## 보기

- A. 인스턴스, 볼륨, 로드 밸런서, 퍼블릭 IP
- B. 인스턴스, 볼륨, 네트워크 인터페이스, 보안 그룹
- C. 인스턴스, 서브넷, 라우팅 테이블, 인터넷 게이트웨이
- D. 인스턴스, 볼륨, Container Registry 이미지, VPC

## 정답

B

## 해설

카카오클라우드 공식 문서에 따르면, Kubernetes Engine 클러스터를 삭제하면 해당 클러스터와 연관된 인스턴스(워커 노드), 볼륨, 네트워크 인터페이스, 보안 그룹이 자동으로 삭제됩니다. 따라서 클러스터 삭제 전 이러한 리소스에 대한 영향을 충분히 검토해야 합니다.

## 오답 포인트

- A: 로드 밸런서와 퍼블릭 IP는 클러스터 삭제 시 자동 삭제 목록에 포함되지 않습니다. 자동 삭제되는 리소스는 인스턴스, 볼륨, 네트워크 인터페이스, 보안 그룹입니다.
- C: 서브넷, 라우팅 테이블, 인터넷 게이트웨이는 VPC 구성 요소로, 클러스터 삭제 시 자동으로 삭제되지 않습니다. 이 리소스들은 사용자가 직접 관리해야 합니다.
- D: Container Registry에 저장된 이미지와 VPC는 클러스터 삭제 시 자동 삭제되지 않습니다. Container Registry와 VPC는 클러스터와 독립적으로 관리됩니다.

## 문제

카카오클라우드 Load Balancing 서비스의 **요청 처리 흐름**으로 옳은 것은?

## 보기

- A. 클라이언트 → 대상 그룹 → 리스너 → 대상
- B. 클라이언트 → 리스너 → 대상 그룹 → 대상
- C. 클라이언트 → 대상 → 리스너 → 대상 그룹
- D. 클라이언트 → 대상 그룹 → 대상 → 리스너

## 정답

B

## 해설

카카오클라우드 공식 문서에 따르면, Load Balancing 서비스의 아키텍처와 작업 흐름은 다음과 같습니다. ①클라이언트가 요청을 보냅니다. ②로드 밸런서는 **리스너(Listener)**를 통해 요청을 수신하고 해당 요청을 **대상 그룹(Target Group)**으로 라우팅합니다. ③대상 그룹은 미리 설정한 알고리즘에 따라 하나의 **대상(Target)**으로 요청을 전달합니다. 따라서 흐름은 클라이언트 → 리스너 → 대상 그룹 → 대상 순서입니다.

## 오답 포인트

- A: 리스너가 먼저 요청을 수신한 후 대상 그룹으로 라우팅합니다. 순서가 틀렸습니다.
- C: 대상이 리스너보다 앞에 올 수 없습니다.
- D: 리스너가 마지막에 오는 구조는 존재하지 않습니다.

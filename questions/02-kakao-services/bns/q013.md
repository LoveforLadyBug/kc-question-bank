## 문제

카카오클라우드 Load Balancing의 **상태 확인(Health Check)** 기능에 대한 설명으로 옳은 것은?

## 보기

- A. 서버의 CPU 사용률이 80%를 초과하면 자동으로 새 서버를 추가한다.
- B. 대상의 상태를 주기적으로 점검하고, 장애가 발생한 서버를 트래픽 분배 대상에서 제외하며 정상 서버로 트래픽을 우회한다.
- C. 모든 서버에 동일한 양의 트래픽을 강제로 분배한다.
- D. 서버 장애 발생 시 운영자에게 알림만 발송하고 트래픽은 계속 전달한다.

## 정답

B

## 해설

카카오클라우드 공식 문서에 따르면, Load Balancing의 **상태 확인(Health Check)** 기능은 대상의 상태를 **주기적으로 점검**하고, 장애가 발생한 서버는 **트래픽 분배 대상에서 제외**하며 **정상 서버로 트래픽을 우회**합니다. 이를 통해 안정적인 무중단 서비스를 유지할 수 있습니다.

## 오답 포인트

- A: 서버 자동 추가는 Auto Scaling 기능이며, 상태 확인 기능의 설명이 아닙니다.
- C: 트래픽을 강제 균등 분배하는 것은 상태 확인 기능이 아닌 로드 밸런싱 알고리즘의 역할입니다.
- D: 장애 서버에도 트래픽을 계속 전달한다면 무중단 서비스를 유지할 수 없습니다.

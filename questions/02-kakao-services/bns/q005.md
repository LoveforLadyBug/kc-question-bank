## 문제

카카오클라우드 VPC 보안 그룹의 동작 방식에 대한 설명으로 옳은 것은?

## 보기

- A. 모든 트래픽을 기본적으로 허용하고, 차단할 정책만 명시적으로 설정하는 블랙리스트(Blacklist) 방식으로 작동한다.
- B. 허용된 정책에 대한 응답 트래픽을 별도로 설정해야 하는 상태 비관리 방식(Stateless)으로 작동한다.
- C. 입력된 정책에 대해서만 허용하는 화이트리스트(Whitelist) 방식으로 작동하며, 허용된 정책에 대한 응답은 별도 설정 없이 허용하는 상태 관리 방식(Stateful)으로 작동한다.
- D. 보안 그룹은 서브넷 단위로만 적용되며, 개별 인스턴스에는 적용할 수 없다.

## 정답

C

## 해설

카카오클라우드 공식 문서에 따르면, 보안 그룹은 입력된 정책에 대해서만 트래픽을 허용하는 화이트리스트(Whitelist) 방식으로 작동합니다. 또한 허용된 정책에 대해서는 별다른 설정 없이 응답을 허용하는 상태 관리 방식(Stateful)으로 제공됩니다. 보안 그룹을 사용하면 IP 주소와 포트 번호를 기반으로 인스턴스로 전송된 패킷을 필터링할 수 있습니다.

## 오답 포인트

- A: 보안 그룹은 블랙리스트 방식이 아닌 화이트리스트(Whitelist) 방식으로 작동합니다. 명시적으로 허용한 트래픽만 허용하고 나머지는 차단합니다.
- B: 보안 그룹은 상태 비관리(Stateless)가 아닌 상태 관리 방식(Stateful)입니다. 따라서 허용된 정책의 응답 트래픽을 별도로 설정할 필요가 없습니다.
- D: 보안 그룹은 서브넷 단위가 아니라 인스턴스(네트워크 인터페이스) 단위로 적용됩니다. IP 주소와 포트 번호를 기반으로 패킷을 필터링합니다.

## 문제

카카오클라우드에서 **클라이언트 IP를 보존하여 전달**해야 하는 요구사항이 있을 때 선택해야 하는 로드 밸런서 유형은?

## 보기

- A. ALB(Application Load Balancer)
- B. NLB(Network Load Balancer)
- C. DSRNLB(Direct Server Return Network Load Balancer)
- D. GLB(Global Load Balancer)

## 정답

C

## 해설

카카오클라우드 공식 문서에 따르면, DSRNLB(Direct Server Return Network Load Balancer)는 TCP, UDP 프로토콜을 지원하며 OSI L4로 동작하고, 클라이언트 IP를 보존하여 전달해야 할 경우 선택합니다. ALB는 HTTP/HTTPS 기반 L7 라우팅에, NLB는 대규모 트래픽에서 빠른 처리 속도가 필요한 경우에 주로 사용됩니다.

## 오답 포인트

- A: ALB는 HTTP, HTTPS 프로토콜을 지원하는 OSI L7 로드 밸런서로, 웹 사이트나 마이크로서비스에 적합합니다. 클라이언트 IP 보존이 주요 목적이 아닙니다.
- B: NLB는 TCP, UDP, TLS 프로토콜을 지원하는 OSI L4 로드 밸런서로, 대규모 트래픽의 빠른 처리가 필요한 경우에 선택합니다. 클라이언트 IP 보존이 주요 특징이 아닙니다.
- D: GLB(Global Load Balancer)는 카카오클라우드에서 제공하지 않는 로드 밸런서 유형입니다.

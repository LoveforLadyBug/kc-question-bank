## 문제

카카오클라우드 Object Storage의 설계 목표로 가장 적절한 설명은?

## 보기

- A. 단일 서버의 로컬 디스크에 데이터를 빠르게 캐싱하기 위해 설계되었다.
- B. 높은 확장성과 안정성을 바탕으로 대용량 데이터를 저장·처리하기 위해 설계되었다.
- C. 여러 서버 간 파일을 실시간으로 동기화하기 위한 프로토콜 계층을 제공한다.
- D. 블록 수준의 고속 입출력으로 낮은 지연 시간을 보장하기 위해 설계되었다.

## 정답

B

## 해설

Object Storage는 높은 확장성(Scalability)과 안정성(Stability)을 핵심 설계 원칙으로 삼아 대용량 데이터 저장 및 처리에 최적화된 서비스입니다.
특정 서버에 종속되거나 프로토콜 계층을 통한 실시간 동기화보다는, 분산 환경에서의 대규모 데이터 보관이 주된 목적입니다.

## 오답 포인트

- A: 로컬 디스크 캐싱은 Object Storage의 목적과 다르며, 분산 저장 서비스는 단일 서버에 종속되지 않습니다.
- C: 파일 실시간 동기화와 프로토콜 계층 제공은 File Storage(NFS 기반)의 특성에 해당합니다.
- D: 블록 수준 저지연 입출력은 Block Storage의 설계 목표이며, Object Storage는 이와 다른 접근 방식을 사용합니다.

## 문제

카카오클라우드 File Storage에서 NFS 프로토콜을 사용하는 이유로 가장 적절한 설명은?

## 보기

- A. 데이터를 블록 단위로 분할하여 고속 랜덤 I/O를 제공하기 위해서
- B. 단일 서버가 외부 인터넷에서 파일을 다운로드하기 위해서
- C. 여러 서버와 애플리케이션이 네트워크를 통해 파일을 공동으로 저장·공유하기 위해서
- D. 객체 형태의 데이터를 키-값 구조로 변환하여 저장하기 위해서

## 정답

C

## 해설

NFS(Network File System)는 분산 파일 시스템을 위한 네트워크 프로토콜로, 여러 서버와 애플리케이션이 동일한 스토리지에 동시에 접근하여 파일을 저장하고 공유할 수 있게 합니다.
File Storage는 이 NFS 프로토콜의 특성을 활용하여 다중 클라이언트 환경에서의 파일 공유 요건을 충족시킵니다.

## 오답 포인트

- A: 블록 단위 고속 랜덤 I/O는 Block Storage의 역할이며, NFS 프로토콜의 목적과 다릅니다.
- B: 인터넷에서의 단순 파일 다운로드는 HTTP/FTP 등의 프로토콜을 사용하며, NFS의 주 사용 목적이 아닙니다.
- D: 키-값 구조 변환과 객체 저장은 Object Storage의 방식이며, NFS 프로토콜과 무관합니다.

## 문제

카카오클라우드 File Storage가 NFS 프로토콜을 채택함으로써 얻는 이점으로 가장 적절한 것은?

## 보기

- A. 데이터를 키-값 형태로 변환하여 검색 속도를 높일 수 있다.
- B. 여러 서버와 애플리케이션이 네트워크를 통해 동일한 파일에 동시에 접근할 수 있다.
- C. 블록 단위 I/O 처리로 데이터베이스 워크로드 성능을 극대화할 수 있다.
- D. 인터넷 구간에서 파일을 암호화하여 안전하게 전송하는 전용 채널을 제공한다.

## 정답

B

## 해설

NFS(Network File System) 프로토콜은 네트워크 상에서 파일 시스템을 공유하는 표준 방식으로, 다수의 서버와 애플리케이션이 동일한 파일에 동시에 접근할 수 있게 합니다.
이를 통해 분산 환경에서 데이터 일관성을 유지하면서 협업적인 파일 읽기·쓰기가 가능해집니다.

## 오답 포인트

- A: 키-값 형태의 데이터 변환 및 검색은 Object Storage의 특성이며, NFS 프로토콜의 이점이 아닙니다.
- C: 블록 단위 I/O 처리는 블록 스토리지 및 iSCSI 계열의 특성으로, NFS 기반 File Storage와 다른 개념입니다.
- D: 인터넷 구간 암호화 전용 채널 제공은 NFS의 기본 기능이 아니며, VPN이나 TLS 같은 별도 보안 레이어의 역할입니다.

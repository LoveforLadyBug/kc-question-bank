## 정답 및 해설

### 1. B
**해설**: BCS의 블록 스토리지는 인스턴스와 독립적인 영구 저장소로, 인스턴스가 종료되더라도 데이터가 보존됩니다. 필요에 따라 인스턴스에 연결(attach)하거나 분리(detach)할 수 있습니다.
이 독립성 덕분에 인스턴스를 교체하거나 삭제할 때도 스토리지에 저장된 데이터를 유지할 수 있어 유연한 데이터 관리가 가능합니다.
**오답 포인트**:
- A: 블록 스토리지는 인스턴스 종료 시 자동 삭제되지 않으며, 영구적으로 데이터를 보존합니다.
- C: 일반적으로 블록 스토리지 볼륨은 하나의 인스턴스에 연결되며, 자동 공유 마운트 기능을 기본 제공하지 않습니다.
- D: 블록 스토리지 용량은 필요에 따라 확장 가능하며, 고정 불변이 아닙니다.

<br>

### 2. B
**해설**: 카카오클라우드 공식 문서에 따르면, 버스터블 인스턴스(t1i)는 CPU 크레딧 기반의 동작 방식을 가지며, 평소에 크레딧을 적립하고 필요 시 버스트하여 성능을 높일 수 있습니다.
이 방식은 평균 CPU 사용률이 낮으면서 간헐적으로 높은 성능이 필요한 워크로드에 비용 효율적으로 적합합니다.
**오답 포인트**:
- A: 버스터블 인스턴스는 항상 최대 성능을 보장하지 않으며, 크레딧 제도가 핵심 동작 원리입니다.
- C: t1i는 GPU 특화 인스턴스가 아니며, GPU 연산은 별도의 GPU 인스턴스 유형에서 지원됩니다.
- D: 버스터블 인스턴스는 스토리지 집약 워크로드 전용 인스턴스가 아니며, 일반 컴퓨팅 용도로 사용됩니다.

<br>

### 3. B
**해설**: 멀티 AZ 구성은 특정 가용 영역에 장애(정전, 재해 등)가 발생하더라도 다른 AZ에서 서비스를 계속 운영할 수 있도록 하여 **고가용성(High Availability)** 을 보장하는 것이 핵심 목적입니다.
BCS에서는 여러 AZ에 인스턴스를 분산 배치함으로써 단일 장애점(SPOF)을 제거하고 서비스 연속성을 유지합니다.
**오답 포인트**:
- A: 동일 AZ 내 통신 최적화는 멀티 AZ의 목적이 아니며, 오히려 AZ 간 통신은 동일 AZ 내부 통신보다 지연이 발생할 수 있습니다.
- C: 인스턴스의 CPU·메모리 수직 확장은 Scale-up에 해당하며, 멀티 AZ 구성과 관계없습니다.
- D: 멀티 AZ는 비용 절감이나 데이터 압축 저장을 위한 구성이 아니라 가용성과 내결함성 향상이 목적입니다.

<br>

### 4. B
**해설**: Bare Metal 서비스는 가상화 환경(하이퍼바이저)이 아닌 하드웨어 리소스를 그대로 단독으로 활용할 수 있는 클라우드 서비스입니다.
가상화 오버헤드 없이 물리 서버의 성능을 온전히 사용할 수 있어, 고성능 컴퓨팅이나 특수 워크로드에 적합합니다.
**오답 포인트**:
- A: Bare Metal은 단독 사용 방식이며, 여러 사용자가 물리 서버를 공유하는 방식이 아닙니다.
- C: Bare Metal은 GPU 없이도 사용 가능하며, 가속 컴퓨팅 전용 서비스가 아닙니다.
- D: Bare Metal은 하이퍼바이저 위에서 동작하지 않으며, 이것이 Bare Metal의 가장 큰 특징입니다.

<br>

### 5. B
**해설**: 카카오클라우드 BCS에서 공인 IP 주소는 별도의 비용이 발생하는 유료 자원입니다.
공인 IP를 BCS 인스턴스에 할당하면 인스턴스 실행 상태와 무관하게 해당 IP에 대한 요금이 청구됩니다.
**오답 포인트**:
- A: BCS 인스턴스에서 공인 IP는 무료가 아니며, 사용량에 따라 요금이 부과됩니다.
- C: BCS 인스턴스에서 공인 IP 요금은 인스턴스 실행 여부가 아닌 IP 할당 여부에 따라 책정됩니다.
- D: 공인 IP는 BCS VPC 생성 시 자동으로 무료 제공되지 않으며, 별도로 신청하고 비용을 지불해야 합니다.

<br>

### 6. C
**해설**: BCS(Beyond Compute Service)는 카카오클라우드의 컴퓨팅 서비스 그룹에 속하는 서비스입니다.
사용자가 필요한 만큼 가상 서버를 생성하고 사용할 수 있도록 지원하는 것이 핵심 역할이며, 이는 컴퓨팅 자원 제공을 목적으로 합니다.
**오답 포인트**:

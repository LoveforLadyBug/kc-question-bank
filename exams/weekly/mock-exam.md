## 정답 및 해설

### 1. C
**해설**: BCS(Beyond Compute Service)는 카카오클라우드의 컴퓨팅 서비스 그룹에 속하는 서비스입니다.
사용자가 필요한 만큼 가상 서버를 생성하고 사용할 수 있도록 지원하는 것이 핵심 역할이며, 이는 컴퓨팅 자원 제공을 목적으로 합니다.
**오답 포인트**:
- A: 네트워킹 서비스 그룹은 VPC, 로드밸런서 등 네트워크 관련 서비스를 포함하며 BCS는 이에 해당하지 않습니다.
- B: 스토리지 서비스 그룹은 Object Storage, Block Storage 등을 포함하며 BCS의 분류가 아닙니다.
- D: 데이터베이스 서비스 그룹은 RDS 등 DB 관련 서비스를 포함하며 BCS와 무관합니다.

<br>

### 2. B
**해설**: HPC와 같은 일부 연산 집약적 워크로드에서는 멀티스레딩(Hyper-Threading)으로 인한 캐시 경합, 메모리 대역폭 경합이 오히려 성능을 저하시킬 수 있어 비활성화가 권장됩니다.
물리 코어당 하나의 스레드만 실행되도록 설정하면 코어 독점 사용이 보장되어 해당 유형의 워크로드에서 더 높은 단일 코어 성능을 얻을 수 있습니다.
**오답 포인트**:
- A: 동시 요청이 많은 웹 서버는 멀티스레딩이 활성화될수록 처리량이 향상되므로 비활성화가 불리합니다.
- C: 병렬 처리 빅데이터 워크로드는 스레드 수가 많을수록 유리하여 멀티스레딩을 유지하는 것이 적합합니다.
- D: 수백 개의 경량 프로세스를 동시에 실행하는 컨테이너 환경에서는 논리 코어 수가 많을수록 유리하므로 멀티스레딩 비활성화가 부적절합니다.

<br>

### 3. B
**해설**: 카카오클라우드 Data Catalog는 Apache Iceberg 테이블을 쉽게 등록하고 메타데이터를 통합 관리할 수 있는 데이터 관리 서비스입니다.
다양한 데이터 소스의 메타데이터를 한 곳에서 관리함으로써 데이터 검색과 거버넌스를 효율화하는 역할을 담당합니다.
**오답 포인트**:
- A: 실시간 스트리밍 파이프라인 구성은 Data Catalog가 아닌 별도의 스트리밍 처리 서비스 영역에 해당합니다.
- C: CPU·메모리 실시간 모니터링은 Monitoring 서비스의 기능으로, Data Catalog와는 관련이 없습니다.
- D: 바이너리 로그 자동 백업은 데이터베이스 복구 서비스의 기능이며 Data Catalog의 역할이 아닙니다.

<br>

### 4. A
**해설**: 카카오클라우드 공식 문서의 빠른 시작 가이드에 따르면, 계정 생성 → 결제 수단 등록 → 사용자 등록 및 조직 역할 추가 → 프로젝트 생성 및 프로젝트 역할 추가 → VPC 생성 순서로 기본 설정을 완료합니다.
결제 수단 등록은 계정 생성 직후에 이루어지며, 서비스 사용을 위한 전제 조건이 되므로 프로젝트나 VPC 생성보다 먼저 수행해야 합니다.
**오답 포인트**:
- B: 프로젝트 생성이 계정 생성보다 앞에 위치하며, 결제 수단 등록 순서가 잘못되었습니다.
- C: VPC 생성이 가장 먼저 수행되는 것으로 잘못 기술되어 있으며, 결제 수단 등록이 마지막에 위치합니다.
- D: 결제 수단 등록이 프로젝트 생성 이후에 수행되는 것으로 순서가 뒤바뀌어 있습니다.

<br>

### 5. B
**해설**: BCS는 Ubuntu, CentOS 등의 Linux 계열과 Microsoft Windows 계열을 포함한 다양한 공식 OS 이미지를 기본 제공합니다.

- D: 클러스터는 VPC 내 다양한 서브넷 구성에서 생성 가능하며, 퍼블릭 서브넷으로만 제한되지 않습니다.

### 15. C
**해설**: BCS(Beyond Compute Service)는 카카오클라우드의 **컴퓨팅 서비스 그룹**에 속하며, 사용자가 필요한 만큼 가상 서버를 생성하고 사용할 수 있는 클라우드 컴퓨팅 환경을 제공합니다.
컴퓨팅 서비스 그룹은 가상 서버 인스턴스, Bare Metal 등 연산 처리를 담당하는 서비스들을 포함합니다.
**오답 포인트**:
- A: BCS는 스토리지를 직접 제공하는 서비스가 아니라, 연산(컴퓨팅) 리소스를 제공하는 서비스입니다.
- B: BCS는 네트워크 서비스가 아니며, VPC·서브넷 등의 네트워크 구성요소는 별도 서비스로 분류됩니다.
- D: 데이터베이스 서비스 그룹은 RDS, NoSQL 등을 포함하며 BCS와는 다른 분류입니다.

### 16. B
**해설**: BCS Bare Metal은 하이퍼바이저(가상화 계층) 없이 물리 서버 자원을 사용자가 단독으로 점유하여 최고의 성능을 발휘할 수 있는 클라우드 서비스입니다.
가상화 오버헤드가 없기 때문에 HPC, GPU 컴퓨팅 등 고성능이 요구되는 워크로드에 적합합니다.
**오답 포인트**:
- A: Bare Metal은 단일 테넌트 방식으로, 물리 서버를 여러 사용자가 공유하지 않습니다.
- C: 경량 하이퍼바이저를 사용한다는 것은 가상화가 존재한다는 의미이므로, Bare Metal의 정의에 위배됩니다.
- D: BCS Bare Metal도 클라우드 콘솔을 통해 온디맨드 프로비저닝이 가능한 것이 클라우드 서비스의 핵심 특징입니다.

### 17. B
**해설**: 카카오클라우드 공식 문서에서는 서비스를 처음 시작하는 가장 기본적인 시나리오로 WEB, APP, DB 세 계층으로 구성된 3-Tier 아키텍처 구축을 안내하고 있습니다.

**오답 포인트**:
- A: 모놀리식 단일 인스턴스 구조는 카카오클라우드가 권장하는 기본 시작 시나리오가 아닙니다.
- C: 서버리스 마이크로서비스 아키텍처는 기본 입문 시나리오가 아닌 고급 구성에 해당합니다.
- D: 쿠버네티스 클러스터 구성은 Kubernetes Engine을 활용한 별도의 시나리오이며 가장 기본적인 시작 시나리오가 아닙니다.

### 18. B
**해설**: 카카오클라우드 Kubernetes Engine은 VPC 기반 위에서 쿠버네티스를 손쉽게 실행할 수 있도록 지원하는 관리형(Managed) 쿠버네티스 서비스입니다.
관리형 서비스이므로 마스터 노드의 운영·관리 부담 없이 클러스터를 생성하고 애플리케이션을 배포할 수 있습니다.
**오답 포인트**:
- A: Kubernetes Engine은 베어메탈 전용이 아닌 VPC 기반의 가상화 환경에서 동작합니다.
- C: 단일 가용 영역 제한은 Kubernetes Engine의 특징이 아닙니다.
- D: 관리형 서비스이므로 사용자가 마스터 노드를 직접 설치하거나 운영할 필요가 없습니다.

### 19. C
**해설**: 카카오클라우드 Object Storage는 코드 기반의 API를 제공하여 사용자가 프로그래밍을 통해 데이터에 접근하고 관리할 수 있도록 지원합니다.
이를 통해 애플리케이션과의 손쉬운 연동이 가능하며, HTTP 기반의 RESTful API를 활용한 유연한 데이터 관리가 가능합니다.

- B: NFS 프로토콜 기반 파일시스템 접근은 File Storage에 해당하는 방식입니다.
- D: SSH 터널링 전용 방식은 Object Storage의 공식 접근 방법이 아닙니다.

### 20. B
**해설**: 카카오클라우드 공식 문서에 따르면, 카카오클라우드를 시작하는 가장 기본적인 시나리오로 WEB, APP, DB로 구성된 3 Tier 아키텍처 구축을 소개합니다.
3 Tier 아키텍처는 프레젠테이션·비즈니스 로직·데이터 계층을 분리하는 전통적이고 보편적인 설계로, 클라우드 환경에서 기본 네트워크 및 서버 구성을 학습하는 데 적합합니다.
**오답 포인트**:

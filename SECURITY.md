# SECURITY.md
# KakaoCloud Essential Basic Course — 문제 유출 방지 정책

> 이 문서는 문제은행의 문제가 스터디 외부로 유출되지 않도록 관리하는 정책을 정의합니다.
> 문제가 유출되면 스터디 효과가 없어집니다. 이 문서는 그것을 막기 위한 최소한의 가드레일입니다.

---

## 1. 보안 등급 분류

문제 파일은 `status` 값에 따라 보안 등급이 다릅니다.

| status | 등급 | 접근 범위 | 설명 |
|--------|------|----------|------|
| `draft` | 내부 | 팀 전체 | AI 초안. 아직 검증 전 |
| `review` | 내부 | 팀 전체 | 리뷰 대기 중 |
| `active` | **제한** | 팀 전체 (시험 전 열람 주의) | 실제 시험에 출제되는 문제 |
| `deprecated` | 내부 | 팀 전체 | 폐기된 문제. 참고용으로만 보존 |

> **핵심 원칙**: `active` 문제의 정답·해설은 시험 직전에는 열람하지 않습니다.
> 스터디 효과를 위해 시험 중에는 문제만 보고 정답을 모르는 상태에서 응시합니다.

## 2. Git 저장소 접근 제어

### 저장소 공개 범위

| 시나리오 | 권장 설정 |
|---------|---------|
| 팀/사내 스터디 전용 | **Private 저장소** (필수) |
| 외부 공개 서비스로 확장 시 | 별도 Public 저장소 분리 후 active 문제 제외 |

> 이 프로젝트는 팀/사내 스터디 목적이므로 **저장소는 반드시 Private** 으로 유지합니다.

### 브랜치 보호 규칙

```
main 브랜치:
  □ 직접 push 금지 (PR만 허용)
  □ PR 머지 조건: 리뷰어 1인 이상 승인
  □ force push 금지

questions/ 디렉토리:
  □ status: active 파일 수정 시 리뷰어 2인 이상 권장
```

---

## 3. 문제 외부 공유 금지 원칙

아래 행위는 금지됩니다.

| 금지 행위 | 이유 |
|-----------|------|
| active 문제를 카카오톡·슬랙 등 외부 채널에 캡처·공유 | 미응시자에게 답 유출 |
| 시험 세트 파일(`exams/`)을 스터디 비참여자에게 공유 | 동일 |
| `questions/` 디렉토리를 공개 URL로 공유 | 저장소 private 원칙 위배 |
| AI 도구(ChatGPT 등)에 active 문제 전문을 붙여넣기 | 외부 서버 저장 위험 |

---

## 4. 시험 세트 파일 관리

`exams/` 디렉토리의 파일은 사용 후 아래와 같이 관리합니다.

```
exams/
├── weekly/
│   ├── week-01.md          ← 사용 완료 → archived/ 로 이동
│   └── week-02.md          ← 현재 사용 중
├── mock-exam-01.md         ← 사용 완료 후 archived/ 로 이동
└── archived/               ← 사용 완료된 시험 세트 보관
    └── week-01.md
```

**시험 세트 재사용 금지**: 동일한 세트를 다른 스터디 기수에서 재사용하면
이전 참여자가 정답을 기억할 수 있으므로, 매번 새로 생성합니다.

## 5. 스터디 중 문제 노출 규칙

| 상황 | 규칙 |
|------|------|
| 주차별 스터디 진행 중 | 해당 주차 문제만 공개, 다음 주차 문제 미리 보기 금지 |
| 모의시험 진행 중 | 정답 파일 별도 관리, 시험 종료 후 공개 |
| 해설 리뷰 세션 | 전체 해설 공유 가능 (이미 시험 완료 후) |

## 6. 유출 사고 대응

만약 active 문제가 외부에 유출됐다고 판단되면:

```
[1] 유출 범위 파악 (어떤 문제, 어느 채널)
[2] 해당 문제 status → deprecated 처리 (즉시)
[3] 대체 문제 생성 요청 (generate-questions.py 또는 수동)
[4] 팀 공지: 해당 문제는 이후 시험 세트에서 제외됨을 안내
[5] 재발 방지 논의 → 이 문서 업데이트
```

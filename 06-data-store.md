### 이 챕터가 다루는 범위

Data Store는 MySQL, PostgreSQL(관계형), Redis(인메모리), MongoDB(문서형) 등
관리형 데이터베이스 서비스를 포함합니다.
각 DB 유형의 특징 및 카카오클라우드 관리형 서비스의 장점을 다룹니다.

## 2. 토픽별 출제 명세

### topic: `mysql` / `postgresql` — 관리형 관계형 DB

| 항목 | 내용 |
|------|------|
| 목표 문제 수 | 4문제 |
| 난이도 분포 | 1×1, 2×2, 3×1 |
| 핵심 개념 | 관계형 DB 특징, 관리형 서비스의 이점(자동 백업, HA 등) |

**반드시 포함할 팩트**
- 관리형 서비스: 패치, 백업, 모니터링을 카카오클라우드가 담당
- RDBMS: 정형 데이터, 트랜잭션, SQL 기반

---

### topic: `redis` — 관리형 Redis

| 항목 | 내용 |
|------|------|
| 목표 문제 수 | 4문제 |
| 난이도 분포 | 1×1, 2×1, 3×2 |
| 핵심 개념 | 인메모리 DB, 캐싱 용도, 낮은 지연시간 |

**반드시 포함할 팩트**
- Redis: 메모리에 데이터를 저장하는 인메모리 DB
- 주요 용도: 캐싱, 세션 관리, 실시간 리더보드
- 디스크 기반 DB 대비 매우 빠른 읽기/쓰기 속도

**출제 예시 방향**
```
난이도 1: "인메모리 데이터 저장으로 빠른 읽기/쓰기를 제공하는 DB 서비스는?"
난이도 3: "API 응답 속도를 높이기 위해 자주 조회되는 데이터를 임시로
           저장하는 패턴에 적합한 서비스는?"
```

---

### topic: `mongodb` — 관리형 MongoDB

| 항목 | 내용 |
|------|------|
| 목표 문제 수 | 3문제 |
| 난이도 분포 | 1×1, 2×1, 3×1 |
| 핵심 개념 | 문서형(Document) NoSQL, 비정형 데이터, JSON 유사 구조 |

**반드시 포함할 팩트**
- MongoDB: 문서(Document) 기반 NoSQL DB
- 스키마가 유연하여 비정형·반정형 데이터에 적합
- RDBMS와 달리 고정 스키마 불필요

---

### topic: `db-comparison` — DB 유형 비교

| 항목 | 내용 |
|------|------|
| 목표 문제 수 | 2문제 |
| 난이도 분포 | 3×1, 4×1 |
| 핵심 개념 | RDBMS vs NoSQL 특징 비교, 사용 시나리오별 선택 기준 |

**핵심 비교표**

| 구분 | MySQL/PostgreSQL | Redis | MongoDB |
|------|-----------------|-------|---------|
| 유형 | RDBMS | 인메모리 | Document NoSQL |
| 데이터 구조 | 정형(테이블) | Key-Value | JSON 문서 |
| 주요 강점 | 트랜잭션, 일관성 | 속도(저지연) | 유연한 스키마 |
| 주요 용도 | 주문, 결제 | 캐싱, 세션 | 로그, 카탈로그 |

### topic: `ha-backup` — 고가용성 및 백업

| 항목 | 내용 |
|------|------|
| 목표 문제 수 | 2문제 |
| 난이도 분포 | 3×1, 4×1 |

카카오클라우드에서 제공하는 API(Application Programming Interface)를 사용하기 위해 수행해야 하는 사전 작업은 다음과 같습니다.

## IAM 액세스 키 발급 ​

IAM 사용자 자격 증명인 IAM 액세스 키(IAM access key)는 IAM 액세스 키 ID와 보안 액세스 키를 의미하며, API 인증 토큰 발급을 위해 필요한 정보입니다.
IAM 액세스 키의 권한은 선택한 프로젝트의 IAM 역할에 따라 결정됩니다. IAM 액세스 키는 콘솔 우측 상단의 프로필 항목에서 발급받을 수 있으며, 자세한 발급 절차는 자격 증명 > IAM 액세스 키 발급 을 참고하시기 바랍니다.

## API 인증 토큰 발급 ​

API 인증 토큰을 발급받기 위해서는 IAM 액세스 키 발급 이 선행되어야 합니다.
기본적으로 API 인증 토큰은 12시간 이후 만료되며, 상황에 따라 12시간 이내라도 변경되거나 만료될 수 있습니다.
2026년 4월 2일 22:00 이후부터, API 인증 토큰 발급 시 Request Header의 'Content-Type' 값을 application/json 으로 반드시 설정해야 합니다.
application/json
적용 시점 이후, Request Header에 'Content-Type: application/json' 이 포함되지 않은 요청은 인증 토큰이 발급되지 않으니, 서비스 이용에 차질 없도록 반드시 사전 조치를 부탁드립니다.
API 인증 토큰은 카카오클라우드 계정 ID와 비밀번호를 대신하는 액세스 인증 토큰입니다. CLI나 API에서 API 인증 토큰을 사용하여 인증하고, 카카오클라우드 서비스를 이용할 수 있습니다. API 인증 토큰을 발급받는 방법은 파라미터 구성에 따라 두 가지 방식으로 구분됩니다.

| 구분 | 필요한 파라미터 |
|---|---|
| API 인증 토큰 발급 방법 1 | IAM 액세스 키 ID, 보안 액세스 키 |
| API 인증 토큰 발급 방법 2 | IAM 액세스 키의 이름, 보안 액세스 키, 사용자 UUID |

### API 인증 토큰의 권한 변경/만료 안내 ​

기본적으로 API 인증 토큰은 발급 후 12시간 이후 만료되며, 상황에 따라 12시간 이내라도 변경되거나 만료될 수 있습니다. 이 경우, 새로운 토큰을 발급받아야 합니다.
API 인증 토큰의 권한은 보안 및 권한 관리를 위해 주기적으로 갱신되거나 변경될 수 있습니다. 따라서 사용자는 필요한 작업을 수행하기 전에 토큰의 유효성을 확인하고 필요한 경우 새로운 토큰을 발급받아야 합니다. 이를 통해 안전하고 효율적으로 API를 활용할 수 있습니다. API 인증 토큰의 권한은 아래 상황에 따라 변경되거나 만료될 수 있습니다.

| 경우 | 설명 |
|---|---|
| 권한 변경 | 소속 프로젝트 역할이 변경된 경우, 토큰 발급 시점과 현재 역할을 비교하여 일치하는 역할 또는 권한만 상속 - 예시: 프로젝트 관리자에서 프로젝트 멤버로 변경될 경우: 프로젝트 멤버 권한을 상속받음 |
| 권한 만료 | Case 1. 프로젝트 역할이 삭제(프로젝트에서 내보내기)된 경우 Case 2. 카카오클라우드 콘솔 > 우측 상단 프로필 > 자격 증명 > IAM 액세스 키 항목에서 사용자가 IAM 액세스 키를 직접 삭제한 경우 |

더 나은 내일을 위한 카카오클라우드의 기술과 혁신 카카오클라우드는 더 빠르고 안전한, 새로운 클라우드 세상을 만들어 가고자 합니다.
클라우드 서비스는 좋은 땅에 멋진 집을 건축하여 임대하는 것과 비슷합니다. ‘카카오클라우드’는 그저 겉으로 보기에 그럴듯한 집이 아닌, 대들보와 주춧돌이 잘 설계되어 수십 년이 지나도 흔들리지 않는 집을 개발해 왔습니다.
잘 건축된 집에 대들보와 주춧돌이 있다면, 클라우드 사업의 본질은 바로 ‘하드웨어’와 ‘네트워크' 에 있습니다. 클라우드 서비스가 잘 되어 있는지 보려면, 이 하드웨어와 네트워크가 잘 설계되어 있는지 질문을 해보면 클라우드의 수준을 볼 수 있을 것입니다.

### 시작부터 다른 클라우드 서비스 ​

클라우드에서 '네트워크(Network)'는 서비스 전체의 기술 수준을 가늠할 수 있는 매우 중요한 기술입니다. 특히, 복잡한 비즈니스 워크로드를 위한 클라우드 서비스를 고려하신다면, 해당 클라우드 서비스가 시작부터 안전하고 유연한 네트워크 환경인지를 반드시 확인해야 합니다.
카카오클라우드 서비스를 시작하기 위한 첫 단계는 논리적으로 격리된 가상 네트워크 환경인 VPC(Virtual Private Cloud) 를 생성하는 것입니다. 카카오클라우드의 컴퓨팅 서비스인 Beyond Compute Service (BCS) 의 경우에도 VPC와 같은 네트워크 연결이 필요합니다.
카카오클라우드에서 사용자는 기본적으로 설정된 VPC 환경을 기반으로 맞춤형 가상 네트워크를 구성할 수 있습니다. 이러한 VPC는 논리적으로 격리되었을 뿐 아니라 다중 AZ(Multi Availability Zone)를 통해 물리적으로 분리되기 때문에 높은 안정성과 가용성을 보장합니다.
멀티/하이브리드 클라우드 환경의 전환과 기존 클라우드 환경의 신속한 확장을 위해서는 네트워크 간 연결 또는 여러 VPC 간 유기적인 연결이 필수적입니다. 하지만 서로 다른 환경을 연결할 경우, 성능 저하와 복잡한 네트워크 설정 등의 문제가 발생할 수 있습니다. 이 경우, 카카오클라우드의 Transit Gateway 서비스를 사용하면 성능 저하 없이 여러 VPC를 유기적으로 연결하고, 데이터 센터에서 사용하는 프라이빗 환경과 유사하게 구성할 수 있습니다. 또한 Transit Gateway는 다수의 VPC 및 온프레미스 네트워크를 연결하여 중앙 집중식 네트워크를 구축할 수 있게 해주며, 다중 AZ 환경을 지원하여 전체 시스템의 가용성을 향상하고 빠른 재해복구(DR, Disaster Recovery) 아키텍처를 구현할 수 있습니다.

### 고성능, 고집적 인프라에 대한 집착 ​

카카오클라우드는 클라우드 시장 진출과 동시에 하드웨어 연구 개발에도 활발히 참여하고 있습니다. 직접 연구 개발을 통해 고성능 및 고집적인 클라우드 기술력을 축적해 왔으며, 이제 그 성과는 카카오클라우드의 우수한 성능으로 명백하게 나타나고 있습니다.
카카오클라우드는 AMD 등 글로벌 선두 기업과의 긴밀한 기술 혁신을 함께 진행해 오고 있습니다. 글로벌 No.1 프로그래머블 반도체(FPGA) 기업인 AMD - 자일링스와의 협력으로 SmartNIC을 공동 개발하고, 카카오클라우드에 탑재되는 고성능 인프라를 연구 개발하여 고성능 클라우드 네이티브 아키텍처를 구현했습니다. 또한, AI 분야 및 고성능 데이터센터 구성을 위하여 글로벌 선두 기업들과 다양한 협력을 이어가고 있습니다.
클라우드 서비스는 끊임없이 진화하고 고객의 요구 변화에 맞춰야 합니다. 고성능 하드웨어를 직접 개발하여 클라우드 서비스에 적용하고 있다는 것은 카카오클라우드가 다른 국내 클라우드 사업자들과는 확실히 차별화된다는 의미이기도 합니다. 카카오클라우드는 다양한 하드웨어 연구 개발에 주력하며, 클라우드 서비스 성능 향상을 위한 지속적인 투자를 진행하고 있습니다.

### 뛰어난 성능과 경제적인 비용의 컴퓨팅 서비스 ​

카카오클라우드는 최신 하드웨어와 최적화된 인프라 구조를 통해 다양한 고성능 컴퓨팅 서비스를 제공합니다. 고객들은 타사 대비 뛰어난 성능과 경제적인 가격으로 카카오클라우드 컴퓨팅 서비스를 이용할 수 있습니다.
현재 BCS 인스턴스의 요금은 경쟁사 대비 약 80~85% 수준으로, 다양한 가격 정책을 통해 추가 할인이 가능합니다. 가상 머신이나 서버 등 구입 요소에 대한 구축 비용이 궁금하시다면, 요금 계산기 를 통해 예상 요금을 확인하시기 바랍니다.
카카오클라우드의 클라우드 컴퓨팅 서비스인 Beyond Compute Service(BCS) 는 비즈니스 특성을 고려하여 최적화된 컴퓨팅, 메모리, 스토리지 및 네트워크를 구성할 수 있는 다양한 인스턴스를 제공합니다. 현재, 인스턴스 유형 으로는 범용(General purpose), 컴퓨팅 최적화(Compute optimized), 메모리 최적화(Memory optimized), 가속 컴퓨팅(Accelerated computing), 비디오 트랜스코딩(Video transcoding) 인스턴스가 있습니다.
카카오클라우드는 최신 하드웨어 확보와 고성능 컴퓨팅 서비스 개발에 주력하여 국내 CSP 중 가장 다양한 GPU(Graphics Processing Unit) 인스턴스를 제공하고 있습니다. 또한, 타사 GPU 서비스의 유사 사양 대비, 80~90% 수준의 경쟁력 있는 요금의 GPU 인스턴스를 제공합니다.

### 강력한 보안 ​

카카오클라우드는 고객의 데이터와 자산을 안전하게 보호하기 위해 강력한 보안 기능을 제공합니다. 상시 이벤트 모니터링, 보안 동향 및 취약점 정보 제공, 보안 관제 월간 보고서 등의 서비스를 통해 고객은 안정적이고 안전한 클라우드 환경에서 비즈니스를 운영할 수 있습니다. 또한, 보안 전문 인력과 다양한 서비스를 제공하여 클라우드 서비스 이용자들에게 높은 수준의 보안을 제공합니다.

### 신속한 기술 지원 ​

클라우드 서비스 이용 중에 발생할 수 있는 장애 및 다양한 문제에 신속하게 대응하기 위해, 전문 기술 인력이 고객 지원 서비스를 제공합니다. 또한 헬프데스크 를 운영하여 보다 빠르고 효율적인 기술 지원을 제공합니다.

### 서비스 이용 대상 ​

클라우드 컴퓨팅은 비즈니스에 있어 많은 장점을 제공하지만, 클라우드 인프라 구축 및 관리는 복잡한 작업일 수 있습니다. 특히 기업이나 공공기관, 중/대형 쇼핑몰 업체, 그리고 글로벌 시장 진출을 목표로 하는 업체에게 더 큰 어려움을 초래할 수 있습니다. 카카오클라우드는 누구나 사용할 수 있는 퍼블릭 클라우드(Public Cloud)이며, 카카오클라우드의 상담 및 도입 문의 를 통해 자세한 상담을 받으실 수 있습니다.
인프라 구축은 비용, 시간, 기술, 리소스 등의 측면에서 많은 어려움이 있습니다. 카카오클라우드에서 제공하는 기술 컨설팅 서비스를 통해, 고객은 기업의 요구 사항과 목표에 맞는 클라우드 인프라 아키텍처를 설계하고 구축할 수 있습니다. 전문 컨설턴트는 기업의 니즈에 최적화된 클라우드 서비스와 인프라 마이그레이션과 관련된 모범 사례를 제시합니다.
보안은 클라우드 서비스에서 가장 중요한 요소 중 하나입니다. 클라우드 서비스를 이용하고 싶지만 보안에 우려가 있는 경우, 카카오클라우드의 기술 컨설턴트는 고객의 보안 요구 사항을 이해하고, 적절한 보안 제어 및 모니터링 메커니즘을 구현할 수 있도록 지원합니다.
클라우드 컴퓨팅은 유연성과 확장성을 제공하여 인프라 리소스를 효율적으로 활용할 수 있도록 도와줍니다. 카카오클라우드의 기술 컨설팅 서비스는 고객의 요구 사항을 분석하고, 최적의 클라우드 리소스를 선택하며, 자동화 및 스케일링 기능을 구현하는 방법을 제시합니다. 이를 통해 업체는 인프라 비용을 최적화하고, 유연하게 리소스를 조정할 수 있으며, 높은 가용성과 성능을 유지할 수 있습니다.
중/대형 쇼핑몰은 많은 트래픽과 거래를 처리해야 하므로 안정적이고 확장 가능한 인프라가 필요합니다. 카카오클라우드의 기술 컨설팅 서비스는 쇼핑몰 업체의 트래픽 패턴과 요구 사항을 이해하고, 성능 최적화, 로드 밸런싱, 캐싱 전략, 데이터베이스 관리 등을 포함한 최적의 아키텍처를 제시합니다. 이를 통해 쇼핑몰 업체는 사용자 경험을 향상하고, 안정적인 거래 처리를 보장할 수 있습니다.
SMS, Push, Email 등 다양한 채널을 활용하여 고객과의 효과적인 커뮤니케이션을 유지함으로써 고객 만족도를 높일 수 있습니다.

## VPC 생성 ​

Virtual Private Cloud(VPC) 는 조직을 위한 전용의 가상 네트워크입니다. VPC는 다른 가상 네트워크와 논리적으로 분리되어 있으며, VM 인스턴스와 같은 클라우드 리소스를 실행할 수 있습니다.
카카오클라우드의 다양한 클라우드 리소스를 사용하기 위해서는, 먼저 VPC 생성 및 설정이 필요합니다.

## 시작하기 전에 ​

카카오클라우드 서비스를 사용하기 위해서는, 먼저 조직의 대표 클라우드 관리자가 회원 가입을 통해 클라우드 조직 계정을 생성해야 합니다. 회원 가입 전 사업자등록증 사본, 연락처 정보(대표자 이름, 전화번호, 주소) 정보를 미리 준비해 주시기 바랍니다.
카카오클라우드의 모든 서비스는 역할 기반 접근 제어(Role Based Access Control, RBAC) 방식의 IAM(Identity and Access Management)을 기반으로, 사용자에게 특정 역할을 부여하여 사용자 / 조직 / 프로젝트를 제어할 수 있는 권한을 설정합니다. 카카오클라우드의 IAM 역할별 상세 권한은 IAM 역할 을 참고하시기 바랍니다.
카카오클라우드 조직 계정을 최초 생성한 사용자는 조직의 최상위 관리 권한을 보유하는 조직 소유자 가 되며, 조직 소유자에게는 조직 관리자 와 빌링 관리자 역할이 자동으로 부여됩니다.
따라서, 이 단계는 조직의 대표 클라우드 관리자 가 수행하시기 바랍니다.

| 상태 아이콘 정의 | 이벤트로 관리하고 있는 서비스 상태를 시각적으로 구분 - 🟡 확인 필요 : 추가 점검이 필요한 서비스 - 🔴 처리 중 : 영향이 확인되어 조치 중인 서비스 - 🟢 해결됨 : 정상 상태로 복구된 서비스 |
